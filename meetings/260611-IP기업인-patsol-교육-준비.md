## 논의 배경

발명진흥원 사업의 일환으로 IP 기업인(일반 IT 기업 IP 부서) 50개 사를 대상으로 PatSol 교육을 진행해야 함. 교육 시작 전 시나리오 정리, 기능 버그 점검, 유저 등급별 메뉴 제어 등을 사전에 준비하는 차원에서 회의 진행.

---

## 주요 논의

### 교육 대상 및 일정
- 참여 기업: 50개 사 (일반 IT 기업 내 IP 부서 담당자, 변리사 아님)
- 기업당 교육 횟수: 3회 (처음/중간/끝 구성으로 추정)
- 교육 방식: 오프라인 방문 위주이나 웨비나 병행 검토
  - 세 번 현장 방문은 현실적으로 어려움 → 첫 회는 오프라인, 이후 웨비나 유도 방향
  - 교육팀 규모: 방문 시 최소 2인 1조로 구성 (질문 수집·답변 역할 분리)
- 사업 기간: 6개월, 한 달 내 몰리지 않도록 일정 분산 필요

### PatSol 교육 시나리오 (한상호님 발표 기준)
IP 업무 흐름을 기업 유형별로 소개한 뒤 PatSol 기능 시연으로 이어지는 구성:

1. **IP 업무 개관** — 권리화, 리스크 관리, 전략 지원 세 축 소개
2. **PatSol 제공 기능** — 선행기술 조사, R&D 방향 설정, 경쟁사 모니터링 (FTO·분쟁·라이선스는 간접 지원)
3. **시연 흐름**
   - 특허 검색 → 결과 엑셀 다운로드
   - 분석 탭 업로드 → 출원 동향, CPC 분포, 기술 흐름도 확인
   - OS 매트릭스(목적-수단 분류) 업로드 및 결과 확인
   - 개별 특허 채팅 질의 / 복수 특허 동시 비교 질의
4. **기업별 맞춤 준비** — 방문 전 해당 기업 검색으로 분석 자료 미리 준비

### 교육 시나리오 준비 방향
- 현재 한상호님이 박람회 시연 프로세스 자료를 정리해 공유 예정 → 이걸 베이스로 시나리오 수정
- 스크립트까지는 불필요하나, 기능별 설명 틀 + 예상 질문 사전 준비 필요
- 질문이 전문 영역 벗어나면 "메일로 확인 후 답변" 방식으로 대응
- 처음 몇 회는 팀원이 함께 가서 질문 패턴을 파악하고 보완

### 기능 버그 점검 / UX 개선 범위
- **UX 개선은 교육 이후 피드백 반영 방향** (교육 전에는 기본 버그 수정만)
- 교육 시나리오 기준으로 버그 점검할 주요 기능:
  - 엑셀 다운로드 → 분석 탭 업로드 흐름
  - 분석 리포트 채팅 질의
  - 백그라운드 작업 전송
- OS 매트릭스 관련 이슈:
  - 목적-수단이 1:1 매핑이므로 목적 수 변경 시 수단 재생성 필요
  - 초과 입력 시 파일이 날아가는 버그 확인 필요
  - 특허가 여러 목적-수단 조합에 중복 포함될 수 있어 총 건수가 300건 초과 가능

### 유저 등급 정책
- **현재 상태**: 변리사 인증 여부(approved/아님)만 체크하며, 기능마다 개별 권한 확인 로직이 흩어져 있음. 청구항 생성 메뉴는 변리사 인증 없으면 진입 차단.
- **신규 필요 사항**: 일반 등급 사용자에게 특정 메뉴·버튼 자체가 보이지 않도록 처리
  - 청구항 생성 버튼 포함한 변리사 전용 기능 묶음을 일반 등급에서 아예 숨기는 방향
  - 이를 위해 유저 등급 체계(변리사 / 일반 등 명시적 등급 컬럼) 신규 설계 필요

### 사전 설문조사
- 교육 전 참여 기업들의 니즈·기대사항을 미리 파악하는 사전 설문조사 진행 제안
- 박람회 때 만난 기업들은 주로 "검색 범위", "OS 매트릭스 원리", "상사 보고용 자료화" 등을 궁금해했음
- 기존 박람회 CRM 기록 공유 예정 (한상호님)

## 결정 사항

1. **교육 전 작업 범위**: UX 개선은 교육 이후, 버그 수정만 우선 진행
2. **교육 방식**: 첫 회 오프라인, 이후 웨비나 병행 / 방문 시 2인 1조
3. **유저 등급 제어**: 일반 등급에서 청구항 생성 등 변리사 전용 기능 묶음 숨김 처리 → 유저 등급 체계 신규 설계 필요
4. **시나리오 베이스**: 한상호님 박람회 시연 자료 기반으로 시나리오 수정
5. **교육 대상 범위**: PatSol 서비스 내에서 진행 (발명진흥원과 변경 시 협의 필요)
