## 4. 결제 흐름 수정 (담당: @이금희 기획화면 수정)

### 결제 방식
- 오픈 초기: 계좌이체/현금 결제 우선
  - 관리자가 입금 확인 후 결제 완료 처리 플로우
- 카드 결제: 심사 지연 가능성으로 후순위

### 서류 안내 단계
- 결제 후 개인/법인 출원에 따라 필요 서류 안내 (신분증, 사업자등록증 등)
- **구현 범위: 전자서명까지 포함** (BizCopilot 캡처 화면 참고)
  - PDF 화면에서 손그림 서명 + **SMS 인증번호 입력** 병행 방식
  - 이 조합이 특허법 시행규칙 제5조의2 요건(서명 이미지 + SMS 인증) 충족
  - 공동인증서는 도입 안 함. SMS 인증까지만 구현
  - SMS 인증 타이밍: 서명 직전 또는 직후 (UX 검토 필요)

### 기타
- 결제화면 버튼 일부 변경 필요 (@이금희)
- 어드민 쪽은 순복님 전담
- 웹/앱 호환성 고려 필요

## 5. 상담 연결 구조 (카카오톡 채널)

- **웹**: QR 코드 → 카카오톡 상담 채널 이동
- **앱**: URL 버튼 → 카카오톡 상담 채널 이동
- 고객이 PatSol 앱/웹 내에서 입력한 정보를 카카오톡으로 전달 가능 여부 검토 필요
  - 현재 uuid는 "유사기술 찾아보기" 클릭 시점부터 생성됨
  - 발명 아이디어 입력 화면에서는 현재 작성 정보를 추적 불가
  - **[기술 검토 필요]** 발명 아이디어 입력 단계도 기억하는 방향 vs. 해당 단계에서 상담 연결 비활성화 (@김순복)

## 8. 관리자 — 변리사 검토 표시 기능

- **목적**: 지재처 및 변리사협회의 AI 안전 활용 규정 대응 (고객 요구 아님)
- **구현**: 관리자 화면에서 변리사가 직접 "검토 완료" 체크
- **적용 범위**: 내부 기록 + 고객 UI 노출 둘 다

### UX 흐름 변경
```
기존: 결제 완료 → 명세서 초안 즉시 다운로드
변경: 결제 완료 → 변리사 검토 대기 → 검토 완료 → 다운로드 가능
```
- 고객 화면에 status 표시 UI 추가 필요
