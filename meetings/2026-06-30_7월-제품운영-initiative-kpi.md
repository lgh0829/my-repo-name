# 7월 제품 운영 Initiative / Epic / KPI 정리

작성일: 2026-06-30

## 배경

6월까지 청구항 구조화, AI Chat Panel, 명세서 작성 화면 디자인 개선 등이 상용에 배포되었다. 방향성은 긍정적이나, 현재는 신규 기능 확장보다 기존 기능이 실제 업무 흐름에서 의도한 대로 작동하는지 검증하는 것이 우선이다.

특히 기존 청구항별 보정 탭 오류, 명세서 작성 화면에서 AI 코파일럿 사용 시 발생하는 문제, AI 응답이 실제 명세서에 반영되는 과정에 대한 점검이 필요하다.

동시에 7월에는 검색·분석 기능으로 전면 전환하기보다, 검색·분석 기반 작업과 고객 피드백 체계, 업데이트 안내 체계, MCP 외부 파일럿 공개 준비를 병행한다.

## 7월 핵심 방향

7월은 검색·분석으로 전면 전환하는 달이 아니라, 기존 명세서 작성 기능의 신뢰성을 확보하면서 검색·분석과 MCP 확장을 위한 기반 및 고객 피드백 체계를 구축하는 달로 설정한다.

핵심 방향은 다음과 같다.

- 명세서 작성 기능 안정화 및 실제 사용 흐름 검증
- 고객 대상 업데이트 안내 및 웹 매뉴얼 운영 체계 구축
- 변리사 피드백 루프 및 가망 고객 유치 기반 마련
- 검색·분석 기능의 기반 작업 진행
- MCP 외부 파일럿 공개 준비

## 상위 Initiative

### Initiative 1. 명세서 작성 제품 신뢰성 확보

목표:

6월까지 배포된 명세서 작성 관련 기능이 실제 업무 흐름에서 안정적으로 작동하는지 검증하고, 핵심 오류를 수정한다.

연결 Epic:

- 명세서 작성 기능 안정화 및 사용 흐름 검증

핵심 KPI:

- 명세서 작성 핵심 플로우 QA 완료율 100%
- P0/P1 수준의 핵심 오류 0건으로 감소
- AI Chat Panel 개선 이슈 재현 및 정리 완료
- 주요 사용 흐름 회귀 테스트 통과율 90% 이상

### Initiative 2. 고객 커뮤니케이션 및 피드백 운영 체계 구축

목표:

제품 업데이트가 고객의 이해, 사용, 피드백으로 이어지도록 릴리즈 안내와 웹 매뉴얼 운영 체계를 구축한다.

연결 Epic:

- 고객 업데이트 안내 및 웹 매뉴얼 운영 체계 구축
- 변리사 피드백 루프 및 고객 검증 체계 구축

핵심 KPI:

- 릴리즈 안내 템플릿 확정
- PatSol, 이메일, SNS 등 채널별 안내 포맷 확정
- 웹 매뉴얼 구조 및 페이지 템플릿 확정
- 변리사 대상 업데이트 안내 메일 발송
- 고객 피드백 및 인터뷰 확보

### Initiative 3. 검색·분석 및 MCP 확장 기반 구축

목표:

검색·분석 기능의 대규모 고도화 전에, 향후 확장을 위한 검색 인프라와 분석 결과 재사용 구조, MCP 외부 파일럿 기반을 구축한다.

연결 Epic:

- 검색·분석 기반 기능 구축
- MCP 외부 파일럿 공개 준비

핵심 KPI:

- 신규 Elasticsearch 적용 완료
- Milvus 연결 완료
- 도면 검색 MVP 동작 확인
- 분석 결과 저장·이력·공유 구조 구축
- MCP 파일럿 범위 확정 및 연결 테스트 완료

## Epic 후보

### Epic 1. 명세서 작성 기능 안정화 및 사용 흐름 검증

목표:

6월까지 배포된 청구항 구조화, AI Chat Panel, 명세서 작성 화면 개선이 실제 업무 흐름에서 안정적으로 작동하는지 검증하고 핵심 오류를 수정한다.

포함 이슈:

- 청구항별 보정 탭 오류 점검 및 수정
- 명세서 작성 전체 플로우 QA
- AI Chat Panel 사용성 점검 및 개선 항목 정리
- AI 응답이 명세서에 반영되는 과정 검증
- 회귀 테스트 체크리스트 작성

KPI 후보:

- 명세서 작성 핵심 플로우 QA 완료율 100%
- 핵심 오류 P0/P1 이슈 0건으로 감소
- AI Chat Panel 개선 이슈 재현 및 정리 완료
- 주요 사용 흐름 회귀 테스트 통과율 90% 이상

### Epic 2. 고객 업데이트 안내 및 웹 매뉴얼 운영 체계 구축

목표:

제품 업데이트가 고객의 이해와 실제 사용으로 이어지도록 릴리즈 공지, 메일, SNS, 홈페이지 매뉴얼의 운영 체계를 표준화한다.

포함 이슈:

- 제품 업데이트 안내 템플릿 정의
- MaSol 기반 릴리즈 운영 체계 정리
- 채널별 안내 기준 정리: PatSol, 이메일, SNS
- 홈페이지 웹 매뉴얼 구조 기획
- 기능별 튜토리얼 콘텐츠 우선순위 선정
- 오류 신고 및 개선 의견 제출 경로 정의

KPI 후보:

- 릴리즈 공지 템플릿 1종 이상 확정
- 채널별 안내 포맷 3종 확정: PatSol, 이메일, SNS
- 웹 매뉴얼 IA 및 페이지 템플릿 확정
- 6월 배포 기능 대상 매뉴얼 작성
- 고객 피드백 접수 경로 1개 이상 오픈

비고:

이 Epic은 7월 기획자가 가장 먼저 착수해야 할 중심 이니셔티브로 본다. 이후 기능 업데이트, 변리사 컨설팅, 가망 고객 유치, MCP 공개까지 일관되게 연결하는 운영 허브 역할을 한다.

### Epic 3. 변리사 피드백 루프 및 고객 검증 체계 구축

목표:

변리사와 가망 고객에게 업데이트 내용을 안내하고, 실제 사용 피드백을 제품 개선으로 연결하는 반복 구조를 만든다.

- 인터뷰 대상자 모집
- 피드백 수집 양식 설계
- 피드백 분류 및 Jira 반영 기준 정의

KPI 후보:

- 변리사 대상 업데이트 메일 발송 1회
- 피드백 응답 확보
- 변리사 인터뷰 진행
- 고객 피드백 기반 개선 이슈 생성
- 가망 고객 상담 또는 컨설팅 연결

### Epic 4. 검색·분석 기반 기능 구축

목표:

검색·분석 기능의 대규모 고도화 전에, 향후 확장을 위한 검색 인프라와 결과 재사용 구조를 먼저 구축한다.

- 최근 작업 및 분석 이력 조회
- 분석 결과 공유 권한 구조 정의

KPI 후보:

## 기획자 우선 착수 Jira Issue 후보

### 1. 제품 업데이트 안내 운영 체계 정의

설명:

제품 업데이트가 발생할 때 고객에게 어떤 내용을, 어떤 채널로, 어떤 형식으로 안내할지 표준 운영 체계를 정의한다. MaSol 릴리즈 기능을 기준으로 PatSol 공지, 이메일, SNS 등 채널별 안내 방식과 작성 템플릿을 정리한다.

주요 작업:

- 릴리즈 공지 기본 템플릿 정의
- 채널별 안내 기준 정리: PatSol, 이메일, SNS
- 안내 시점 기준 정의: 배포 전, 배포 후, 핫픽스, 파일럿
- 고객 유형별 안내 범위 정리: 기존 고객, 변리사, 가망 고객, MCP 파일럿 사용자
- MaSol에서 릴리즈 노트를 작성·발행하는 운영 흐름 정리

완료 기준:

- 업데이트 안내 템플릿 초안 작성 완료
- 채널별 발행 기준 정리 완료
- MaSol 기반 릴리즈 운영 플로우 정리 완료

### 2. 웹 매뉴얼 및 튜토리얼 운영 구조 기획
