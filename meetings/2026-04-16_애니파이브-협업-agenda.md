# 애니파이브 협업 협의 — Agenda

- 날짜: 2026-04-16 (이번주 내 답변 요함)
- 참석자:

---

## 1. 회의 목적

애니파이브로부터 협업 제안이 접수됨. PatSol의 기능을 IPBrain 플랫폼 내에 탑재하는 방안을 검토하고, 대응 가능 여부 및 타임라인을 이번 주 내로 확정하여 회신.

---

## 2. 핵심 논의

### 1) Agenda Item 1 — 직무발명서 생성 기능 제공 가능 여부

**논의 포인트**
- PatSol이 연구자(기업 내 발명자) 대상 직무발명서 초안 생성을 지원할 수 있는가?
- 직무발명서에 포함되는 데이터의 형태: 이미지 유형(사진/캡처/구조도 등), 텍스트 유형, 또는 파일 여러 개
- 현재 명세서 작성 지원 기능과의 기술적 중첩/차별점은?
- 직무발명서 특유의 항목(발명의 경위, 공헌도, 선행기술 유무 등)을 커버하는 데 추가 개발이 필요한가?

**결정 필요 사항**
- [ ] 기능 지원 가능 여부 (Yes / No / 조건부)
- [ ] 이미지 입력(사진·캡처·구조도) 처리 가능 여부 — 현재 멀티모달 지원 범위 확인 (추가개발, 통합 일정이 충분할 때 가능)
- [ ] 여러 개 파일 처리 가능 여부 — 현재 멀티모달 지원 범위 확인 (추가개발, 통합 일정이 충분할 때 가능)
- [ ] 기존 명세서 기능 대비 추가 개발이 필요한 항목 목록화
- [ ] 가능하다면 MVP 제공 가능 시점

---

### 2) Agenda Item 2 — IPBrain 내 통합 탑재 범위 협의

애니파이브가 요청한 통합 범위:
1. 직무발명서 생성
2. 선행기술조사 실행
3. 명세서 생성

**논의 포인트**
- IPBrain이 어디에 구축되어 있는지 (고객회사별 온프레미스? 애니파이브 내 Private 클라우드? Public 클라우드(AWS, GCP, Azure)?)
- 각 기능의 API/모듈 단위 제공 가능 여부: 각 기능별로 받아야 하는 데이터 및 리턴해줄 수 있는 데이터 형태 확인
- 수익 모델(용역 및 유지보수, 라이선스, 사용량 기반 등) 에 대한 언급 있었는지?
- 애니파이브 측이 통합 일정을 어느 정도로 보고 있는지

**결정 필요 사항**
- [ ] IPBrain 인프라 환경 파악 → 연동 방식 확정 (REST API 제공 / 온프레미스 배포 / 기타)
- [ ] 제공 가능한 기능별 Input/Output 명세 초안 작성 (회신 전 내부 정의 필요)
- [ ] 수익 모델 방향 내부 정렬 — 애니파이브 언급 여부와 무관하게 우리 안 먼저 확정
- [ ] 애니파이브에 회신할 조건 및 담당자

---

### 3) Agenda Item 3 — 메일 회신 시 질의할 사항 확정

**배경**
이번주 회신은 가능 여부 + 조건 수준으로 충분하나, 본 협의 진입을 위해 사전 확인이 필요한 항목을 회신 메일에 함께 질의해야 함.

**질의 후보 항목**

| 항목 | 질의 이유 |
|------|-----------|
| IPBrain 인프라 환경 (온프레미스 / 프라이빗 / 퍼블릭 클라우드) | 연동 방식 자체가 달라짐 |
| 예상 고객사 수 및 동시 사용자 규모 | 아키텍처·과금 설계에 영향 |
| 출시 목표 시점 | 우리 일정 가능 여부 판단 기준 |
| 수익 모델에 대한 애니파이브 측 의견 | 협상 포지션 사전 파악 |

**향후 본 협의 진입 시 요청할 문서 — 기술 통합 요구사항서**
- 기능 요구사항: 각 기능별 입력 데이터, 기대 출력 형태, 품질 기준
- 기술 제약: 사용 언어/프레임워크, 인증 방식, 응답 시간 요건
- 보안/컴플라이언스: 고객사 특허 데이터 처리 범위, 데이터 저장 위치 제한
- 통합 일정: 마일스톤별 기대 산출물
- 운영 요건: SLA, 장애 대응, 유지보수 범위

**결정 필요 사항**
- [ ] 회신 메일에 포함할 질의 항목 최종 선정
- [ ] 기술 통합 요구사항서 요청 시점 결정 (회신 메일에 포함 vs. 킥오프 미팅 이후)
