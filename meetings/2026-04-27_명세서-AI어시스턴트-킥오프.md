# 명세서 AI 어시스턴트 — 킥오프 회의록

> 2026-04-27 | 참석: 김순복, 이윤주, 배성현, 민건호, 한상호

---

## 2. 핵심 논의 — 에이전트 구조

### 1) 배경/이슈

- AI 기능을 구현할 때 모든 도구를 단일 function calling으로 통합할지, 역할별로 에이전트를 분리할지 결정 필요
- 청구항 생성과 명세서 생성은 맥락과 도구 셋이 다르므로 일괄 처리 시 품질 저하 우려

### 2) 제안

- A안: function calling 단일 통합 — 구조 단순, 관리 편리하나 역할 구분이 불명확해질 수 있음
- B안: sub-agent 방식으로 기능별 에이전트 분리 + function calling 혼합
- 우려사항: 에이전트 수가 늘어날수록 컨텍스트 분산 위험
- 추가 아이디어: 개별 에이전트가 하나의 컨텍스트를 공유하는 구조로 분산 문제 해소

### 3) 결정 사항

- ✅ 최종 결정: Sub-agent + function calling 혼합 방식 채택. 청구항 생성 / 명세서 생성 각각 개별 에이전트로 분리하되, 모든 에이전트가 하나의 컨텍스트를 공유
- 📌 보류 사항: 기재요건 검토를 별도 에이전트로 분리할지 여부 → PoC로 검증
- 🔎 추가 검토 필요: function calling 1개로 통합 vs 기재요건 검토 별도 에이전트화 기준 정의

## 3. 핵심 논의 — 모델 선정 및 AB 테스트

### 1) 배경/이슈

- 기능마다 요구되는 추론 수준이 다르므로 동일 모델을 일괄 적용하면 비용 대비 비효율 발생
- 의도분류·읽기 도구(추론 중심)와 쓰기·텍스트 생성(출력 중심)의 최적 모델이 다를 수 있음

### 2) 제안

| 안 | 의도분류기 / 읽기 도구 | 쓰기 / 텍스트 생성 |
|---|---|---|
| A | 좋은 모델 (추론) | 낮은 모델 |
| B | 낮은 모델 | 좋은 모델 |

- 우려사항: 어느 쪽이 실제 품질에 더 영향을 주는지 사전 판단 어려움
- 추가 아이디어: config로 모델만 교체할 수 있도록 에이전트 구현 → 테스트 비용 절감

### 3) 결정 사항

- ✅ 최종 결정: AB 테스트로 최적 조합 결정. 시나리오 실행 후 입력/결과 쌍을 문서로 공유하여 품질 평가
- 📌 보류 사항: 의도분류기 구현 참고 모델(Codex, Claude) 최종 선택 → 테스트 결과 후 결정
- 🔎 추가 검토 필요: 람다 AB 테스트 개발 범위 및 일정 확정

## 4. 핵심 논의 — 채팅 범위 (MVP 기준)

### 1) 배경/이슈

- MVP 범위를 최소화하되, 실사용 시 컨텍스트 과부하 문제가 발생할 수 있어 사전 설계 필요
- 채팅이 길어질수록 컨텍스트 누적 → 응답 품질 저하 및 비용 증가 위험

### 2) 제안

- A안: 명세서당 채팅 1개로 제한, 요약/clear 기능은 추후 고도화
- B안: MVP부터 컨텍스트 요약/clear 기능을 설계에 포함
- 우려사항: 요약/clear 정책(시점, 트리거 조건)이 미정 상태에서 구현 시작 시 설계 변경 비용 발생

### 3) 결정 사항

- ✅ 최종 결정: MVP는 명세서당 채팅 1개. 컨텍스트 요약/clear 기능은 MVP 설계에 포함 (추후 고도화가 아닌 초기부터 고려)
- 📌 보류 사항: 새 채팅 시작 기능 → 추후 고도화
- 🔎 추가 검토 필요: 채팅 컨텍스트 요약/clear 정책 정의 (요약 시점, 트리거 조건)

## 5. 핵심 논의 — 기획 보완 사항 (FRD 반영)

### 1) 배경/이슈

- 킥오프 과정에서 기존 FRD에 누락되거나 구체화가 필요한 항목 식별
- 개발 착수 전 FRD에 반영하지 않으면 구현 범위 혼선 발생 가능

### 2) 제안

- 추가 아이디어: 선택지 카드(C-04)에 자유 입력 옵션 추가하여 사용자 의도 표현 범위 확장
- 추가 아이디어: 명세서 최초 생성 완료 시 챗 패널 첫 메시지로 요약·검토·근거 자동 제공

### 3) 결정 사항

- ✅ 최종 결정: 아래 4개 항목을 FRD에 반영

| 항목 | 내용 | FRD 영향 |
|---|---|---|
| 선택지 카드 자유 입력 | C-04에 사용자가 직접 원하는 방향을 입력할 수 있는 옵션 추가 | C-04 스펙 수정 |
| 최초 생성 시 챗 첫 메시지 | 명세서 최초 생성 완료 후 AI 챗 패널 첫 메시지로 요약·검토·근거 제공 | P-9 포맷 구체화 |
| 초안 생성 요약 기능 | 1단계에서 AI 요약·검토 기능을 러프하게 마련 | Phase 1 범위 업데이트 |
| 채팅 요약 / clear | MVP 단계부터 설계에 포함 | 미결 사항 OQ 추가 필요 |

- 📌 보류 사항: 각 항목의 구체적 스펙은 이윤주가 FRD 업데이트 후 별도 리뷰
- 🔎 추가 검토 필요: 채팅 요약/clear OQ 정책 정의
