토큰 정리 결정 뷰

남은 lint error 3건을 시각적으로 비교합니다. 대비비는 이 페이지에서 WCAG 공식으로 직접 계산합니다. 파란 테두리 = 하네스 추천안.

① helper 텍스트 대비 (현행 3.54:1, 본문 기준 4.5:1 미달)

흰 배경 위 helper 텍스트. 작은 본문 텍스트는 AA 4.5:1 기준입니다.

선택지: A 현행유지+예외 · B gray650 #5E5E5E · C 신규 #737373

② danger 버튼 (흰 텍스트, 현행 배경 #F54848 = 3.56:1)

버튼 라벨(굵은/큰 텍스트)은 WCAG상 3.0:1(UI/Large) 기준 적용이 가능합니다. 아래는 실제 버튼 렌더와 두 기준 통과 여부입니다.

선택지: A 현행유지(UI/Large 3.0 기준 → 통과) · B 더 어두운 빨강(4.5 통과) · C 현행유지+예외

③ gray250 / gray200 스케일 역전 (ΔE<1, 거의 동일)

gray 스케일은 숫자가 클수록 어두워야 하나, gray250(#EEEEEE)이 gray200(#EDEDED)보다 밝습니다. 아래에서 실제로 구분되는지 확인하세요.

선택지: A 통합(gray250 제거, border.light→gray200) · B 예외등록(값 유지) · C 값 보정(gray250를 더 어둡게)

두 큰 견본을 나란히 두었습니다. 경계가 보이지 않으면 사실상 중복입니다.
