# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with this repository.

> 이 문서는 Claude Code(claude.ai/code)가 이 저장소에서 작업할 때 참고하는 지침서입니다.

## 목적 (Purpose)

이곳은 현실의 문제를 Claude를 활용해 빠르게 해결하기 위한 개인 실험 공간입니다. 각 하위 폴더는 독립적인 실험으로, 자체적인 환경과 목표를 가집니다.

두 가지 핵심 동기:
1. 빠른 반복(iteration)을 통해 현실 문제의 실용적인 해결책을 찾는다.
2. Claude 공식 문서를 살펴보며 Claude를 최대로 활용하는 방법을 발견한다.

## 구조 (Structure)

루트 디렉터리에는 이 지침 문서만 둡니다. 모든 실험은 하위 폴더에 두며, 각 폴더는 독립적으로 구성합니다.

```
cc-lab/
├── CLAUDE.md          ← 이 문서 (루트 실험의 취지·방향만 기록)
└── <실험-이름>/       ← 실험 하나당 폴더 (독립 실험 환경)
    ├── 실험 목표를 적은 README 또는 메모
    └── (실험별 파일과 환경)
```

새로운 실험을 시작할 때는 새 하위 폴더를 만듭니다. 실험 간에 관심사를 섞지 않습니다.

## 작업 방식 (Working in this repo)

- **각 하위 폴더는 독립적입니다.** 루트에는 빌드 스크립트, 의존성, 설정이 없습니다.
- **각 실험 폴더의 로컬 CLAUDE.md를 확인하세요.** 실험별 명령어, 설정 단계, 아키텍처 메모가 들어있습니다.
- **완성도보다 속도.** 일회성 실험이므로, 재사용성보다 빠르고 직접적인 해결을 우선합니다.
- **루트에는 설치·테스트 명령어가 없습니다.** 항상 실험 폴더로 이동해 그 폴더의 지침을 따르세요.
- **응답 언어:** 사용자와의 대화는 한국어로 합니다.

## 실험 종료 및 정리 (Cleanup on Experiment Deletion)

사용자가 특정 실험 폴더의 삭제를 요청하면, 폴더를 지우기 전에 반드시 아래 절차를 따릅니다.

### 1. 설치 추적
실험 폴더의 CLAUDE.md 및 대화 이력을 참조해 해당 실험을 위해 설치한 항목을 목록화합니다.
- `brew install` / `brew install --cask` 로 설치한 패키지
- `npm install -g` 로 설치한 글로벌 패키지
- `pip install` 등 기타 패키지 매니저로 설치한 항목

### 2. 의존성 충돌 확인
삭제 전에 다른 실험 폴더나 시스템에서 같은 패키지를 사용하는지 확인합니다.
- 다른 실험 CLAUDE.md에서 동일 패키지 언급 여부 검토
- `brew uses --installed <패키지>` 로 역의존성 확인

### 3. 삭제 실행
충돌이 없는 패키지만 제거합니다.
```bash
brew uninstall <패키지>          # brew 패키지
brew uninstall --cask <패키지>   # brew cask 패키지
npm uninstall -g <패키지>        # npm 글로벌 패키지
brew autoremove                  # 고아 의존성 정리
```

### 4. 폴더 삭제
패키지 정리 후 실험 폴더를 삭제합니다.
```bash
rm -rf <실험-폴더>
```

> **원칙:** 실험 종료 후 시스템을 실험 시작 전 상태로 되돌린다. 설치 이력이 불분명한 경우 사용자에게 확인 후 진행한다.
