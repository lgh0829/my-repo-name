# 2026년 7월 PatSol 뉴스레터 기획

## 1. 뉴스레터 운영 목적

뉴스레터를 단순한 릴리즈 공지 수단이 아니라, 마케팅 홍보에 동의한 사용자와 지속적으로 접점을 만드는 채널로 운영한다.

- 완성도가 검증된 기능은 이용 가치와 사용 방법을 명확히 소개한다.
- 아직 보완 중인 기능은 핵심 메시지로 앞세우지 않고 `출시 예정` 또는 `강화 예정` 수준으로만 다룬다.
- 청구항 기재불비 보정 등 완성도 검증이 더 필요한 기능은 7월호의 핵심 메시지에서 제외한다.
- 릴리즈 정보만 전달하지 않고 PatSol 활용 팁, 특허 AI 활용 사례, 특허 검색·분석 업무 효율화 방법을 지속적으로 다룬다.

## 2. 7월호 1차 대상

- 직무: 변리사
- 상태: 회원가입을 완료했으나 로그인 이력이 없는 사용자
- 수신 조건: 마케팅 홍보 수신 동의

이번 호의 일차 목표는 최신 기능을 많이 알리는 것이 아니라, 사용자가 PatSol의 안정적인 핵심 기능에서 첫 업무 가치를 경험하도록 만드는 것이다.

## 3. 확정된 핵심 경험

가입 직후 별도 설정이나 복잡한 준비 없이 다음 흐름을 경험하도록 안내한다.

```text
발명 내용 입력
↓
유사특허 검색
↓
검색 결과 확인
↓
검토할 주요 문헌 선별
```

- 핵심 기능: 특허 검색·분석
- 콘텐츠 방향: 기능 목록보다 실제 업무에서 얻는 결과를 먼저 설명
- CTA 방향: 검색·분석 업무를 바로 시작할 수 있는 구체적인 행동 문구 사용

## 4. 1회차 주제 후보

### 우선 후보

**신규 출원 전 선행기술조사를 PatSol로 시작하는 방법**

- 변리사가 보편적으로 이해할 수 있는 업무 상황이다.
- 발명 내용을 입력하고 유사특허를 확인하는 짧은 흐름으로 설명할 수 있다.
- 아직 보완 중인 명세서 기능을 전면에 내세우지 않아도 PatSol의 가치를 보여줄 수 있다.
- 제품 화면 캡처와 단계별 활용 팁을 함께 제공하기 좋다.

### 보조 콘텐츠 후보

- 검색 결과에서 먼저 검토할 문헌을 좁히는 방법
- 특허 AI 검색 결과를 검토할 때 확인할 항목
- 선행기술조사 시간을 줄이는 입력문 작성 팁
- 실제 검색·분석 활용 사례

## 5. 7월호 권장 구성

1. 이번 호 소식 한눈에 보기
2. 신규 출원 전 선행기술조사에서 반복되는 어려움
3. 발명 내용으로 유사특허를 찾는 기본 흐름
4. 검색 결과를 검토할 때 확인할 항목
5. PatSol에서 검색 시작하기 CTA

## 6. 표현 원칙

- `AI가 변리사의 업무를 대신한다`는 인상을 주지 않는다.
- 변리사가 판단의 주체이고 PatSol은 검색·분석 업무를 지원하는 도구로 표현한다.
- 확인되지 않은 시간 절감률이나 정확도 수치를 사용하지 않는다.
- 기능명보다 고객이 수행할 수 있는 업무와 얻는 결과를 먼저 설명한다.
- 본문은 짧게 쓰고 상세 내용은 CTA 링크에서 확인하도록 한다.

## 7. 미결 사항

- [ ] `신규 출원 전 선행기술조사`가 실제 고객에게 가장 자주 시연·안내한 대표 업무인지 확인
- [ ] 7월호에 사용할 실제 제품 화면 캡처 선정
- [ ] 검색 시작 CTA의 최종 연결 URL 확인
- [ ] 대상 사용자 명단과 마케팅 수신 동의 상태 확인
- [ ] 발송일과 내부 검토 담당자 확정
- [ ] 실제 활용 사례 또는 고객 문의 사례 확보

## 8. 다음 대화에서 이어갈 질문

> 실제 고객에게 PatSol을 보여줄 때 `신규 출원 전 선행기술조사` 외에 더 자주 설명하거나 반응이 좋았던 검색·분석 업무가 있었는가?

이 답변을 바탕으로 7월호의 최종 주제, 제목, 섹션별 메시지와 CTA를 확정한다.
