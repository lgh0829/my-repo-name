# 기존 PatSol 고객 이메일 콘텐츠 분류 체계

## 1. 적용 범위

- 고객군: 기존 PatSol 고객
- 채널: 이메일 뉴스레터
- 고객 역할: 변리사, 기업 IP 담당자
- 제외: 가망 고객, PatSol 인앱 공지, SNS 콘텐츠

## 2. 콘텐츠 유형

| 유형 | 발행 목적 | 발행 조건 | 기본 템플릿 | 주요 CTA |
|---|---|---|---|---|
| 월간 종합 | 지난달 주요 변화와 활용 정보를 정기적으로 전달 | 월 1회 | 기본(1) | 사용 방법 보기, 의견 보내기 |
| 기능 추가 | 새로 사용할 수 있는 기능과 이용 방법 안내 | 고객이 사용할 수 있는 새 기능 배포 | 매거진(2) | 새 기능 사용하기 |
| 기능 개선 | 기존 기능의 달라진 점과 고객 이점 안내 | 사용 흐름·성능·품질 개선 | 기본(1) | 달라진 기능 확인하기 |
| 핫픽스 | 문제와 조치 결과를 투명하게 안내 | 고객 영향이 있는 긴급 수정 | 심플(4) | 현재 상태 확인하기 |
| 파일럿 | 제한된 고객의 참여와 의견 수집 | 참여 대상과 운영 기간이 확정된 경우 | 카드형(3) | 참여 신청하기, 의견 보내기 |

## 3. 역할별 발송 원칙

동일한 기존 고객이라도 이용 가능 기능에 따라 변리사와 기업 IP 담당자를 구분한다.

| 역할 | 안내 가능한 기능 범위 |
|---|---|
| 변리사 | 특허검색, 특허분석, 청구항 설계, 명세서 작성 지원 |
| 기업 IP 담당자 | 특허검색, 특허분석 |

- 특허검색·분석 콘텐츠는 두 역할에 공통 발송할 수 있다.
- 청구항 설계·명세서 작성 지원 콘텐츠는 변리사 기존 고객에게만 발송한다.
- 월간 종합호에 변리사 전용 기능이 포함되면 역할별 사본을 만든다.
- 기업 IP 담당자용 사본에서는 이용할 수 없는 기능을 삭제하고, 검색·분석 관련 콘텐츠로 대체한다.

## 4. 공통 작성 원칙

- 가장 중요한 소식을 첫 번째 섹션에 배치한다.
- 섹션은 3~5개, 각 본문은 2~3문장을 기본으로 한다.
- 기능명보다 고객이 얻는 변화와 사용 상황을 먼저 설명한다.
- 확인되지 않은 수치나 효과를 사용하지 않는다.
- 버튼 문구는 `자세히`보다 `새 기능 사용하기`, `사용 방법 보기`처럼 행동을 명확히 표현한다.
- 명세서 관련 표현에서는 변리사가 업무의 주체이며 PatSol은 작성 업무를 지원하는 도구임을 유지한다.
- 이미지 URL은 외부에서 공개적으로 접근할 수 있을 때만 사용한다.

## 5. 발행 전 확인

- 대상 그룹이 기존 고객인지 확인
- 변리사·기업 IP 담당자의 기능 이용 범위 확인
- 캠페인 ID 중복 여부 확인
- 모든 자리표시자와 예시 URL 교체
- 미리보기에서 제목, 이미지, 버튼 링크 확인
- 대상 확인 후 실제 발송 체크
