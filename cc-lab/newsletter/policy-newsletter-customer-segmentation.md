# 뉴스레터 고객 분류 정책

## 1. 목적

PatSol 뉴스레터의 수신자를 기능 이용 자격과 고객 단계에 따라 구분하고, 각 고객군에 적합한 콘텐츠 범위와 CTA를 일관되게 적용한다.

## 2. 분류 기준

고객 유형은 다음 두 축을 조합하여 분류한다.

1. **직무·기능 이용 자격**
   - 변리사
   - 기업 IP 담당자
2. **고객 단계**
   - 기존 고객
   - 가망 고객

| 직무·기능 이용 자격 | 기존 고객 | 가망 고객 |
|---|---|---|
| 변리사 | 전 기능 업데이트, 고급 활용법, 업무 흐름 개선 안내 | 전체 업무 흐름, 도입 효과, 체험·상담 안내 |
| 기업 IP 담당자 | 특허검색·분석 업데이트와 활용법 | 특허검색·분석의 업무 효율, 도입 사례, 상담 안내 |

## 3. 기능 안내 범위

### 3.1 변리사

- 특허검색
- 특허분석
- 청구항 설계
- 명세서 초안 생성 및 명세서 작성 지원

변리사 대상 콘텐츠에서는 특허검색부터 분석, 청구항 설계, 명세서 작성 지원까지 이어지는 전체 업무 흐름을 안내할 수 있다.

### 3.2 기업 IP 담당자

- 특허검색
- 특허분석

기업 IP 담당자는 청구항 설계와 명세서 초안 생성 자격이 없으므로, 해당 기능을 현재 사용할 수 있는 기능처럼 안내하지 않는다.

## 4. 뉴스레터 시스템 입력 기준

현재 뉴스레터 시스템의 단일 `group` 필드로 네 고객군을 구분할 수 있도록 조합형 그룹을 사용한다.

| role | group | 의미 |
|---|---|---|
| 변리사 | `attorney-customer` | 기존 고객 변리사 |
| 변리사 | `attorney-prospect` | 가망 변리사 |
| 기업 IP 담당자 | `corporate-ip-customer` | 기존 기업 IP 고객 |
| 기업 IP 담당자 | `corporate-ip-prospect` | 가망 기업 IP 고객 |

- `role`: 콘텐츠 범위를 결정하는 직무·기능 이용 자격
- `group`: 실제 발송 대상을 선택하는 고객군

## 5. 콘텐츠 운영 원칙

- 변리사 대상 콘텐츠는 PatSol의 전체 기능과 기능 간 연결 흐름을 다룬다.
- 기업 IP 담당자 대상 콘텐츠는 특허검색·분석 기능에 집중한다.
- 동일한 소식이라도 기존 고객과 가망 고객의 도입문과 CTA를 구분한다.
- 매월 네 개의 원고를 각각 작성하지 않고, 직무별 본문 2종을 기본으로 작성한 뒤 고객 단계에 따라 제목·도입문·CTA를 변형한다.
- 변리사 대상 표현에서도 변리사가 업무의 주체임을 유지하고, PatSol은 전문 업무를 지원하는 도구로 표현한다.

## 6. CTA 기준

| 고객 단계 | 권장 CTA |
|---|---|
| 기존 고객 | 새 기능 사용하기, 사용 방법 보기, 의견 보내기 |
| 가망 고객 | 서비스 살펴보기, 체험 신청하기, 도입 상담하기 |

## 7. 캠페인 ID 규칙

`YYYY-MM-{직무}-{고객단계}` 형식을 사용한다.

- `2026-07-attorney-customer`
- `2026-07-attorney-prospect`
- `2026-07-corporate-customer`
- `2026-07-corporate-prospect`
