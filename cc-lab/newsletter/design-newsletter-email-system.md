# PatSol 이메일 뉴스레터 디자인 시스템

## 1. 목적과 범위

기존 PatSol 고객에게 발송하는 이메일 뉴스레터의 시각적 일관성과 제작 효율을 확보한다. 공통 이메일 셸과 재사용 가능한 콘텐츠 컴포넌트를 정의하고, 기본·매거진·카드형·심플 템플릿은 같은 요소의 배치만 달리한다.

- 고객군: 기존 PatSol 고객
- 채널: 이메일 뉴스레터
- 기준 디자인: PatSol 디자인 시스템의 색상·타이포그래피·간격·모서리 토큰
- 기준 시안: `prototype-newsletter-monthly-existing-customer.html`

## 2. 디자인 원칙

1. **한눈에 이해되는 정보 위계**: 제목, 핵심 변화, 이용 방법, CTA 순서가 즉시 드러나야 한다.
2. **하나의 시각 언어**: 발행 유형이 달라도 헤더, 버튼, 카드, 푸터는 동일한 규격을 사용한다.
3. **이메일 호환성 우선**: 1열 구조와 테이블 레이아웃을 기본으로 하고 복잡한 효과와 배경 이미지를 사용하지 않는다.
4. **고객 이점 우선**: 기능명보다 고객 업무에서 달라지는 점을 더 크게 보여준다.
5. **모바일 우선 검수**: 작은 화면에서 텍스트와 CTA가 줄바꿈되더라도 정보 위계가 유지되어야 한다.

## 3. 이메일용 디자인 토큰

제품 디자인 토큰을 이메일에서 안전하게 사용할 수 있는 범위로 축소한다.

| 용도 | 토큰 | 값 |
|---|---|---|
| 브랜드·Primary CTA | `brand-primary` | `#3361BE` |
| CTA 보조 강조 | `chart-blue-800` | `#284C95` |
| 강조 배경 | `background-accent` | `#F3F6FC` |
| 보조 강조 배경 | `background-accent-muted` | `#EBF0FA` |
| 바깥 배경 | `gray-100` | `#F5F5F5` |
| 본문 배경 | `white` | `#FFFFFF` |
| 제목 | `text-primary` | `#1E1E1E` |
| 본문 | `text-body` | `#494949` |
| 보조 설명 | `text-subtle` | `#757575` |
| 기본 테두리 | `border-default` | `#EDEDED` |
| 성공 | `success-500` | `#22C55E` |
| 경고 | `warning-500` | `#FFAB03` |
| 오류·긴급 | `danger-action` | `#F54848` |

### 색상 사용 제한

- 한 화면의 주 강조색은 Primary Blue 한 가지로 제한한다.
- 성공·경고·오류 색상은 상태 안내에만 사용하고 장식 목적으로 사용하지 않는다.
- 본문 텍스트는 배경과 충분한 대비를 유지한다.
- 변리사와 기업 IP 담당자를 색상으로 구분하지 않는다.

## 4. 타이포그래피

```css
font-family: Pretendard, "Apple SD Gothic Neo", "Noto Sans KR", Arial, sans-serif;
```

| 용도 | 크기 / 행간 | 굵기 |
|---|---|---|
| 뉴스레터 제목 | 26px / 36px | 600 |
| 섹션 제목 | 18px / 27px | 600 |
| 도입문 | 15px / 25px | 400 |
| 본문 | 15px / 24px | 400 |
| 버튼 | 14px / 20px | 600 |
| 라벨·보조 설명 | 12~13px / 18~21px | 500~600 |

- 제품 화면의 Display 스타일은 이메일에서 사용하지 않는다.
- 모바일에서 뉴스레터 제목은 23px / 32px로 축소한다.
- 본문 한 문단은 2~3문장으로 제한한다.
- 가운데 정렬은 심플 공지와 히어로 영역에만 사용한다.

## 5. 공통 셸

| 영역 | 규격 |
|---|---|
| 전체 캔버스 | `#F5F5F5`, 가로 100% |
| 본문 컨테이너 | 최대 640px, 흰색, 가운데 정렬 |
| 데스크톱 좌우 여백 | 32px |
| 모바일 좌우 여백 | 18px |
| 섹션 간격 | 32~40px |
| 카드 내부 여백 | 24px |
| 카드 모서리 | 12px |
| 버튼 높이 | 최소 44px |
| 대표 이미지 | 가로형, 권장 1200×500px |

### 공통 구조

```text
숨김 프리헤더
└─ 이메일 캔버스
   └─ 640px 본문
      ├─ 브랜드 헤더 + 발행월
      ├─ 콘텐츠 유형 라벨
      ├─ 제목 + 도입문
      ├─ 대표 이미지(선택)
   ├─ 이번 호 소식 요약
   ├─ 콘텐츠 섹션 3~5개
      ├─ VoC 요청 블록
      └─ 발신자·문의·수신거부 푸터
```

## 6. 공통 컴포넌트

### 6.1 브랜드 헤더

- 왼쪽에 `PatSol. 뉴스레터`
- 오른쪽에 발행월 또는 `제품 업데이트`
- 흰 배경과 하단 1px 테두리 사용

### 6.2 콘텐츠 유형 라벨

- Primary Blue 텍스트와 Accent 배경을 사용한 작은 캡슐 형태
- 권장 문구: `월간 소식`, `새 기능`, `기능 개선`, `조치 완료`, `파일럿`

### 6.3 히어로

- 제목과 도입문을 우선 배치하고 대표 이미지는 그 아래에 둔다.
- 제목은 고객이 얻는 변화를 설명한다.
- 제목에 업데이트 유형을 반복하지 않는다.

### 6.4 이번 호 소식 요약

- 히어로 바로 아래에 3~4개 소식을 한 줄씩 요약한다.
- 독자가 전체 내용을 빠르게 훑고 관심 있는 소식을 찾도록 돕는다.
- 별도의 CTA를 넣지 않고 본문 섹션으로 자연스럽게 연결한다.
- Accent 배경과 12px radius를 사용하되 장식은 최소화한다.

### 6.5 콘텐츠 섹션

- 섹션 하나에 메시지 하나만 담는다.
- 이미지, 제목, 본문, CTA 순서를 유지한다.
- 월간 종합호는 카드 테두리보다 얇은 구분선으로 소식을 나눈다.
- 본문은 핵심 요약 1문단과 보충 설명 1문단 이내로 구성한다.
- 가장 중요한 소식에만 큰 캡처 또는 대표 이미지를 사용한다.

### 6.6 제품 화면 캡처

- 캡처는 카드 제목 위에 배치하고 카드 본문 폭 전체를 사용한다.
- 캡처 프레임은 10px, 바깥 카드 프레임은 12px radius를 사용한다.
- 캡처 프레임과 내부 이미지 사이에 별도 배경색 여백을 만들지 않는다.
- 권장 비율은 16:10, 권장 원본 크기는 1200×750px이다.
- 전체 화면보다 설명하는 기능 영역을 중심으로 잘라낸다.
- 공개적으로 접근 가능한 이미지 URL만 사용한다.
- 개인정보, 고객명, 출원번호, 공개 전 데이터가 포함되지 않도록 검수한다.
- 이미지가 표시되지 않아도 내용을 이해할 수 있도록 본문 설명을 유지한다.

### 6.7 CTA

- Primary CTA는 Primary Blue 배경과 흰색 텍스트를 사용한다.
- 한 섹션에 Primary CTA는 하나만 둔다.
- 버튼 문구는 `자세히` 대신 `새 기능 사용하기`, `사용 방법 보기`처럼 행동을 명확히 쓴다.
- 모바일에서는 버튼을 본문 폭 전체로 확장할 수 있다.

### 6.8 VoC 요청 블록

- Accent 배경을 사용해 일반 카드와 구분한다.
- 한 번에 질문 하나만 제시한다.
- CTA는 `의견 보내기`로 통일한다.

### 6.9 푸터

- 발신자, 답장 주소, 문의 링크, 수신거부 링크를 포함한다.
- 13px 보조 텍스트와 연한 회색 배경을 사용한다.

## 7. 템플릿별 배치

| 시스템 템플릿 | 디자인 배치 | 적용 콘텐츠 |
|---|---|---|
| 기본(1) | 상단 소식 요약 후 섹션을 세로 배열하고 구분선 사용 | 월간 종합, 기능 개선, 여러 변경사항 |
| 매거진(2) | 큰 히어로 이미지와 서사형 본문 | 주요 기능 추가, 고객 사례 |
| 카드형(3) | 독립 카드 3~5개를 1열 배치 | 파일럿 안내, 사례 모음 |
| 심플(4) | 가운데 정렬 제목, 상태 요약, 단일 CTA | 핫픽스, 점검 공지 |

## 8. 이메일 구현 제약

- 레이아웃은 HTML 테이블을 기본으로 한다.
- 핵심 스타일은 인라인 CSS로 제공한다.
- 배경 이미지, 복잡한 그림자, 그라디언트 의존 표현을 사용하지 않는다.
- 이미지는 폭 100%, 높이 자동으로 처리하고 대체 텍스트를 제공한다.
- 링크는 밑줄 또는 버튼 형태로 명확하게 구분한다.
- 열람률은 참고 지표로 보고 CTA 클릭을 주요 성과 지표로 사용한다.

## 9. 검수 기준

- 640px 데스크톱과 360px 모바일에서 가로 스크롤이 없음
- 제목, 본문, CTA 순서가 이미지 없이도 이해됨
- 모든 버튼의 클릭 영역이 최소 44px
- CTA 문구만 읽어도 이동 목적을 알 수 있음
- 기능 이용 자격과 발송 그룹이 일치함
- 발신자·문의·수신거부 정보가 포함됨
