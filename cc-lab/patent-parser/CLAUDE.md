# patent-parser

## 실험 목표

특허 명세서 docx를 도면 평가 에이전트(`patent-figure-reviewer`)가 읽기 쉬운 구조화 Markdown으로 변환한다.

## 핵심 파일

| 파일 | 역할 |
|---|---|
| `parse_patent.py` | docx → 구조화 Markdown 변환 스크립트 |

## 의존성

```bash
pip3 install python-docx
```

## 사용법

```bash
# 기본: <파일명>_parsed.md 로 자동 저장
python3 parse_patent.py 명세서.docx

# 출력 파일 직접 지정
python3 parse_patent.py 명세서.docx output.md
```

## 출력 구조

| 섹션 | 내용 |
|---|---|
| 1. 발명의 명칭 | 특허 제목 |
| 2. 청구항 | 번호별 분리된 청구항 전문 |
| 3. 구성요소 참조번호 목록 | 부호설명 + 본문 추출 조합 테이블 |
| 4. 도면의 간단한 설명 | 도면별 설명 원문 |
| 5. 도면별 검토 체크리스트 | `patent-figure-reviewer` 에이전트 입력용 `- [ ]` 형식 |
| 6. 부호의 설명 원문 | 파싱 근거 원문 |
| 7. 청구항 전문 | 에이전트 참조용 코드블록 |

## 파싱 규칙

- 섹션 헤더 키워드: "청구항", "발명의 상세한 설명", "도면의 간단한 설명", "부호의 설명"
- 참조번호 추출 우선순위: 부호의 설명 섹션 → 본문 `명칭(번호)` 역방향 패턴 → 청구항 단순 등장
- 도면별 체크리스트: 각 도면 설명에 등장한 참조번호만 필터링
