# 청구항·검색·챗봇 스프린트 계획

## 업무 분장

| 담당자 | 역할 |
|---|---|
| 이윤주 | 청구항 기능 FE 구현 |
| 민건호 | 청구항 백엔드 구조화 |
| 배성현 | 재검색 |
| 김순복 | 챗봇 |

## 작업 순서

청구항 구조화하여 write 페이지에서 표시 먼저 구현 → 청구항/명세서 통합

## 이번 스프린트 작업 범위

- 청구항 구조화 조회 (독립항·종속항 계층 구조 표시)
- 독립항 추가
- 종속항 추가
- 청구항 개별 AI 보정
- 청구항 삭제
- 청구항 일괄 AI 보정
- 재검색 / 검색 히스토리 조회 / 토픽 필터 재검색
- 챗봇

> **제외 (우선순위 2):** 청구항 정렬 순서 변경 (번호 재계산·어두 업데이트 포함)

## 세부 Task

### S1 (~4/10)

#### 민건호 — 청구항 BE 구조화

| # | Task | 비고 |
|---|------|------|
| 1 | LLM 파싱 프롬프트 설계 — 독립항/종속항 구분, 어두(preamble) 분리, 인용 관계 추출 | |
| 2 | 파싱 결과 정규화 — 번호·어두·body 필드로 정제, 오파싱 예외 처리 | |
| 3 | claim 테이블 스키마 확정 — `claim_no`, `claim_type(independent/dependent)`, `parent_claim_no`, `preamble`, `body` | |
| 4 | 청구항 계층 조회 API — `GET /documents/{docId}/claims` → 독립항·종속항 트리 구조 반환 | 이윤주 FE 선행 의존 |

#### 배성현 — 재검색 / 검색 히스토리 BE

| # | Task | 비고 |
|---|------|------|
| 1 | 재검색 BE 로직 — 기존 검색 결과 기반으로 조건(키워드·필터) 변경 후 재실행 | |
| 2 | `search_history` 테이블 스키마 확정 — `query`, `filters`, `searched_at`, `result_count` 등 | |
| 3 | 검색 실행 시 히스토리 자동 저장 로직 연동 | |
| 4 | 히스토리 조회 API — `GET /search/history` (페이지네이션, 최신순 정렬) | |

#### 김순복 — Documents 리뉴얼 착수 / 챗봇 착수

| # | Task | 비고 |
|---|------|------|
| 1 | Documents 페이지 현행 구조 분석 및 리뉴얼 레이아웃 초안 작성 | |
| 2 | 페이지 컴포넌트 분리 구조 설계 (섹션별 컴포넌트 트리) | |
| 3 | 챗봇 UI 컴포넌트 초기 구조 설계 (입력창·말풍선·스크롤 영역) | |
| 4 | 챗봇 BE 연동 방식 결정 (REST vs WebSocket, streaming 여부) | 배성현·민건호와 합의 필요 |

### S2 (~4/24)

#### 이윤주 — write 페이지 청구항 구조화 FE / Documents 청구항 섹션

| # | Task | 비고 |
|---|------|------|
| 1 | 청구항 계층 구조 렌더링 — 독립항·종속항 들여쓰기, 계층 트리 표시 | 민건호 조회 API 선행 |
| 2 | 독립항 추가 UI — 빈 독립항 행 삽입, 번호 자동 부여 | |
| 3 | 종속항 추가 UI — 부모 항 선택, 어두(「제N항에 있어서」) 자동 삽입 | BE 어두 스펙 사전 합의 |
| 4 | 청구항 삭제 UI — 단일 삭제, 종속 관계 경고 모달 | |
| 5 | Documents 페이지 청구항 섹션 — 청구항 목록 표시, 섹션 레이아웃 연동 | |

#### 민건호 — 청구항 추가·삭제 API / AI 보정 API

| # | Task | 비고 |
|---|------|------|
| 1 | 독립항 추가 API — `POST /documents/{docId}/claims` (claim_type: independent) | |
| 2 | 종속항 추가 API — `POST /documents/{docId}/claims` (claim_type: dependent, parent_claim_no 포함) | 어두 자동 삽입 로직 포함 |
| 3 | 청구항 삭제 API — `DELETE /claims/{claimId}`, 종속 항 cascade 또는 예외 처리 | |
| 4 | 개별 AI 보정 API — `POST /claims/{claimId}/refine` (LLM 호출 → 보정 결과 반환) | |
| 5 | 일괄 AI 보정 API — `POST /documents/{docId}/claims/batch-refine` (순차 LLM 호출, 진행 상태 반환) | |

#### 배성현 — 토픽 필터 재검색

| # | Task | 비고 |
|---|------|------|
| 1 | 토픽 필터 파라미터 스펙 정의 — 필터 타입·값 목록 구조 확정 | FE와 사전 합의 |
| 2 | 토픽 필터 적용 재검색 BE 로직 구현 | |
| 3 | 토픽 필터 재검색 API — `POST /search/refine` 또는 기존 검색 API에 filter 파라미터 추가 | |

#### 김순복 — Documents 리뉴얼 완료 / 챗봇 MVP

| # | Task | 비고 |
|---|------|------|
| 1 | Documents 페이지 리뉴얼 구현 완료 — 신규 레이아웃·컴포넌트 적용 | |
| 2 | 챗봇 UI 구현 — 입력창, 말풍선, 스크롤 자동 이동 | |
| 3 | 챗봇 BE 연동 — API 호출·응답 렌더링, 로딩 상태 처리 | |
| 4 | 챗봇 MVP 검증 — 기본 질의응답 시나리오 동작 확인 | |
