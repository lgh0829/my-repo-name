# Ubiquitous Language — PatSol 도메인 용어 사전

> 이 파일은 PatSol 제품 설계·개발·커뮤니케이션에서 사용하는 공식 용어를 정의합니다.
> FRD·스펙·회의에서 아래 용어를 일관되게 사용하고, 비표준 표현은 "피해야 할 별칭"을 참조하세요.

---

## 특허 출원 프로세스

| 용어 | 정의 | 피해야 할 별칭 |
|---|---|---|
| **선행기술조사** | 발명의 신규성·진보성 판단을 위해 관련 기존 특허·논문을 탐색하는 작업 | 사전조사, 특허 검색 (조사 행위를 지칭할 때) |
| **청구항** | 특허권의 권리범위를 법적으로 정의하는 핵심 문서. 독립항과 종속항으로 구성 | 클레임 (공식 한국어 문서에서) |
| **명세서** | 발명의 명칭·요약·청구범위·도면 설명·발명의 설명을 포함한 전체 출원 문서 | 스펙 (특허 문서 맥락에서) |
| **독립항** | 발명의 필수 구성요소 전체를 포함하는 최광의 청구항 | 메인 클레임, 기본 청구항 |
| **종속항** | 독립항 또는 다른 종속항을 인용하여 구성요소를 한정·추가하는 청구항 | 서브 클레임 |
| **다중 종속항** | 복수의 부모 청구항을 동시에 인용하는 종속항 (`제N항 또는 제M항에 있어서`) | 복합 종속항 |
| **젭슨 청구항** | 전제부(공지 기술) + "있어서" + 특징부(신규 요소) 구조의 청구항 | — |
| **권리범위** | 특허 청구항이 법적으로 포괄하는 기술적 범위 | 특허 범위, 커버리지 |
| **회피 설계** | 선행기술 청구항과 겹치지 않도록 구성요소를 재정의하거나 상위 개념화하는 전략 | 우회 설계 |
| **거절이유통지** | 심사관이 신규성·진보성 흠결 등을 출원인에게 통보하는 심사 절차 | 거절 통보, 심사 의견 |

## 검색 및 분석

| 용어 | 정의 | 피해야 할 별칭 |
|---|---|---|
| **검색식** | IPC/CPC 분류코드와 키워드 불리언 조합으로 구성된 구조화 검색 쿼리 | 쿼리, 검색어 (검색식 맥락에서) |
| **토픽** | AI가 사용자의 검색 의도에서 추출한 기술 분야 단위. 검색식 생성의 기반 | 카테고리, 태그, 키워드 |
| **토픽 카드** | 추출된 토픽을 사용자가 확인·선택하는 UI 컴포넌트 | 필터 칩, 태그 카드 |
| **비교발명 분석** | 선행특허와 내 발명의 구성요소를 대비하여 차이점을 분석하는 작업 | 특허 비교, 대비 분석 |
| **OS 매트릭스** | 기술 공백(Opportunity Space)을 X·Y 축으로 시각화한 분석 도구 | 기회 매트릭스, 공백 분석 |
| **유사도 스코어** | 벡터 검색 모델이 계산한 검색 쿼리와 특허 문서 간의 의미적 유사도 수치 | 관련도, 매칭 점수 |
| **IPC/CPC** | 국제(IPC) 및 협력(CPC) 특허 분류 코드. 기술 분야를 계층적으로 분류 | 특허 분류, 코드 |

## PatSol 제품

| 용어 | 정의 | 피해야 할 별칭 |
|---|---|---|
| **워크스페이스** | 하나의 발명 프로젝트를 담는 최상위 컨텍스트 단위. 검색 이력·청구항·명세서를 포함 | 프로젝트, 케이스 |
| **직무발명서** | 기업 내 발명자가 발명 내용을 정리한 내부 문서. PatSol의 자연어 검색 입력 소스 중 하나 | 발명 신고서 |
| **초안** | AI가 변리사의 입력을 기반으로 자동 생성한 청구항 또는 명세서의 1차 결과물 | AI 생성물, 자동 완성 결과 |
| **선행기술 Import** | 외부 DB(KIPRIS, Google Patents 등)에서 검색한 특허를 PatSol로 불러오는 기능 | 업로드, 추가 |

---

## 사용자 및 역할

| 용어 | 정의 | 피해야 할 별칭 |
|---|---|---|
| **변리사** | PatSol의 주 사용자. 특허 출원 전 과정을 담당하는 전문직 | 특허 전문가, 사용자 (역할 지칭 시) |
| **특허 담당자** | 기업 IP 팀 소속으로 선행기술조사를 주로 담당하는 비변리사 사용자 | IP 담당자 |
| **발명자** | 발명을 창출한 연구자·엔지니어. PatSol의 간접 이해관계자 | — |

## 예시 대화 (용어 사용 기준)

> **기획자:** "검색 결과에서 사용자가 원하는 문서를 바로 선택할 수 있나요?"
> **설계자:** "**토픽 카드**를 선택하면 해당 토픽 기반의 **검색식**이 자동 생성되고, 결과 목록에서 **선행기술 Import**로 바로 **워크스페이스**에 불러옵니다."
> **기획자:** "선택하지 않은 **토픽**의 특허는 완전히 제외되나요?"
> **설계자:** "아니요, 미선택 **토픽**은 순위가 내려갈 뿐 **권리범위** 검토에서 누락되지 않습니다."

## Flagged — 결정 필요 항목

| 용어 | 현황 | 쟁점 |
|---|---|---|
| **세션** vs **워크스페이스** | 혼용 중 | UI 레이어에서 "세션"이 "워크스페이스"와 구분되는지 정의 필요 |
| **리뷰** vs **검토** | 혼용 중 | 변리사의 초안 확인 행위를 어느 쪽으로 통일할지 결정 필요 |
