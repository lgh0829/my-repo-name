## 1. 개요

### 1.1 목적

변리사가 초기 검색 결과를 확인한 뒤 조건을 조정하여 재검색하고, 동일 프로젝트 내에서 이전 검색 내역을 관리·조회할 수 있는 구조를 제공한다.

### 1.2 배경

현재 PatSol 검색은 세션당 1회 검색만 가능하며 재검색 수단이 없다. 이로 인해:

- 결과가 기대에 못 미칠 때 변리사가 개입할 수단이 없음
- 이전 검색 조건과 결과를 비교·참조할 수 없음
- 토픽 필터 선택 후 검색 주제를 다시 좁히거나 확장하는 흐름이 없음

## 2. 용어 정의

### 2.1 포함 관계

> 현재 작업에서 '검색 주제'는 구현하지 않음

```
검색 히스토리
    └── 검색 주제
          └── 검색 버전
                ├── 검색 유형: 초기 검색
                ├── 검색 유형: 재검색
                └── 검색 유형: 토픽검색

검색 버전 ──참조──▶ 비교발명 (별도 객체)
```

### 2.2 용어 정의

| 용어 | 정의 |
| --- | --- |
| **검색 히스토리** | 누적된 검색 주제 및 검색 버전 전체 목록 |
| **검색 주제** | 검색 히스토리의 상위 그룹 단위. 발명 주제 기준으로 묶이며 하위에 여러 검색 버전을 포함 |
| **검색 버전** | 한 번의 검색 실행으로 생성되는 단위. 검색 유형(초기 검색, 재검색, 토픽검색)·쿼리·결과 건수·일시를 포함 |
| **초기 검색** | 검색 주제 내 최초로 실행되는 검색 버전. 원문 쿼리를 직접 입력하여 실행 |
| **재검색** | 기존 검색 버전의 쿼리를 수정하여 동일 검색 주제 내에서 새로운 검색을 실행하는 행위 |
| **토픽검색** | 검색 결과의 토픽 필터를 선택하여 동일 검색 주제 내에서 새로운 검색을 실행하는 행위 |

## 3. 정책 및 규칙

| # | 정책 | 내용 |
| --- | --- | --- |
| P-1 | 검색 버전 자동 저장 | 검색 실행 완료 즉시 비동기로 저장. 검색 응답과 무관하게 실행 |
| P-2 | 재검색 실행 | [재검색] 클릭 시 프롬프트 편집 상태로 전환. 내용 수정 후 실행하면 현재 검색 맥락 기반으로 새 검색 버전 생성 |
| P-3 | 토픽검색 실행 | 토픽 필터 선택 후 [토픽 검색하기] 클릭 시 확인 모달 노출. 확인 시 현재 검색 맥락 + 선택 토픽을 포함하여 새 검색 버전 생성 |
| P-4 | 비교발명 연관 | 비교발명은 검색 버전과 연관되나 하위 객체가 아닌 별도 객체로 동작 |
| P-5 | 검색 유형 분류 | 모든 검색 버전은 `초기 검색` / `재검색` / `토픽검색` 중 하나로 분류되어 저장 |
| P-6 | 검색 버전 선택 상태 | 검색 실행 후 검색 버전 목록에서 최신 버전이 자동 선택됨. 초기 검색·재검색·토픽검색 모두 동일하게 적용 |

### 4-1. 재검색 (IA 2-1-4)

**위치:** GNB > 특허 검색 > 검색 결과 화면 — 프롬프트 영역

**설명:** 변리사가 현재 검색 결과를 보고 조건이 부족하다고 판단할 때, 프롬프트를 수정하여 새 검색을 실행한다.

### 구성 요소

| 요소 | 설명 |
| --- | --- |
| 프롬프트 편집 모드 전환 | 현재 검색 쿼리를 편집 가능한 상태로 전환. 기존 입력값이 그대로 표시됨 |
| 프롬프트 편집 모드 취소 | 프롬프트 영역 바깥쪽을 클릭하여, 디폴트 상태로 전환. 편집 전 기존 입력값이 그대로 표시됨 |
| [검색] 버튼 | 프롬프트 편집 모드에서 [검색] 버튼 클릭 시 새 검색 실행 |

### 동작 흐름

```
검색 결과 화면
    ↓
GNB의 [재검색] 클릭
    ↓
프롬프트 편집 상태로 전환
  - 기존 쿼리 표시 (수정 가능)
    ↓
변리사가 쿼리 수정 또는 제안 선택
    ↓
검색 실행 → 새 검색 버전 생성 (검색 유형: 재검색)
    ↓
결과 화면 진입 + 자동 저장 (P-1) + 최신 버전 자동 선택 (P-6)
```

---

### 4-2. 토픽검색 (IA 2-2-2)

**위치:** GNB > 특허 검색 > 검색 결과 화면 — 토픽 필터 영역

**설명:** 검색 결과에서 AI가 자동 분류한 토픽 필터를 선택하여 동일 검색 주제 내에서 새로운 검색을 실행한다.

### 구성 요소

| 요소 | 설명 |
| --- | --- |
| 토픽 검색하기 버튼 | 토픽 선택 후 명시적으로 검색 실행을 트리거하는 버튼 |
| 검색 안내 모달 | 토픽 검색 시 검색 결과 변경에 대한 확인 모달 |

### 동작 흐름

```
검색 결과 화면 (토픽 필터 영역)
    ↓
토픽 필터 칩 선택 / 해제
    ↓
예상 결과 건수 실시간 업데이트
    ↓
[토픽 검색하기] 클릭
    ↓
확인 모달 노출
  - 현재 검색 맥락 + 선택 토픽 키워드 표시
  - [취소] / [검색] 버튼
    ↓
확인 시 → 현재 검색 맥락 + 선택 토픽 포함한 새 검색 버전 실행 (검색 유형: 토픽검색)
    ↓
결과 화면 진입 + 자동 저장 (P-1) + 최신 버전 자동 선택 (P-6)
```

### 토픽 검색하기 버튼 활성화 조건

| 조건 | 버튼 상태 |
| --- | --- |
| 토픽 칩 선택 발생 | enabled |
| 토픽검색 실행 직후 | disabled |

---

### 4-3. 검색 히스토리 조회 (IA 2-3-3)

**위치:** GNB > 검색 히스토리

**설명:** 누적된 검색 주제 및 버전을 조회한다.

### 계층 구조

```
검색 히스토리
    └── 검색 주제 (상위)
          └── 검색 버전 1  [초기 검색]  [2026-04-06 14:32]  [43건]
          └── 검색 버전 2  [토픽검색]   [2026-04-06 15:10]  [18건]
          └── 검색 버전 3  [재검색]     [2026-04-06 15:22]  [11건]
```

### 검색 버전 카드 구성

| 요소 | 설명 |
| --- | --- |
| 검색어 표시 | 초기 검색: 원문 쿼리 앞 40자 / 재검색: refined 키워드 / 토픽검색: 적용 토픽 칩 |
| 검색 유형 레이블 | `초기 검색` / `재검색` / `토픽검색` |
| 실행 일시 | `YYYY-MM-DD HH:mm` |
| 결과 건수 | `N건` |

### 동작

| 액션 | 결과 |
| --- | --- |
| 검색 버전 클릭 | 해당 검색 결과 화면으로 이동 |
| 검색 주제 접기/펼치기 | 하위 검색 버전 목록 토글 |
