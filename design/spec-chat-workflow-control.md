# 채팅-워크플로우 연동 에이전트 기술 검토

> 상위 문서: [[ia_spec-v0.1]] — 추가 검토 필요 항목 참고
> 범위: 우측 채팅창 AI가 좌측 워크플로우 패널을 읽고 제어하는 에이전트 구조

---

## 1. 원하는 기능 요약

단순 Q&A 챗봇이 아닌 **워크플로우 파트너** 에이전트.

| 기능 | 설명 |
|---|---|
| 패널 상태 인지 | AI가 현재 적용된 필터·정렬·결과 목록을 컨텍스트로 인지 |
| 액션 제안 | 대화 흐름에서 필터 적용·정렬 변경 등을 사용자에게 제안 |
| 확인 후 실행 | 사용자가 CTA를 클릭하면 AI가 좌측 패널을 직접 제어 |
| 세션 연속성 | 검색 의도 요약을 기반으로 세션 내 대화 맥락 유지 |

---

## 2. 핵심 아키텍처

### 공유 상태 Store + Tool-Use Agent

```
┌─────────────────────────────────────────────────────────┐
│                    공유 상태 Store                       │
│  {                                                      │
│    searchQuery: "자율주행 전기차",                       │
│    appliedFilters: ["배터리 열관리"],                    │
│    sortOrder: "similarity",                             │
│    results: [ patent1, patent2, ... ],                  │
│    selectedPatents: ["KR10-2023-0147821"]               │
│  }                                                      │
└────────────┬──────────────────────┬─────────────────────┘
             │ 읽기 (구독)           │ 쓰기 (dispatch)
             ▼                      ▼
┌────────────────────┐   ┌─────────────────────────────┐
│   좌측 워크플로우   │   │       우측 채팅창             │
│   (상태 구독·렌더)  │   │   (상태 읽기 + 액션 요청)    │
└────────────────────┘   └─────────────────────────────┘
```

## 3. Tool 정의

AI가 사용할 수 있는 도구 목록.

```python
tools = [
    {
        "name": "get_panel_state",
        "description": "현재 좌측 패널의 필터·정렬·결과 상태를 조회",
        "input_schema": {}
    },
    {
        "name": "propose_topic_filter",
        "description": "토픽 필터 적용을 사용자에게 제안. 확인 후 apply_topic_filter 호출",
        "input_schema": {
            "topics": ["string"],   # 제안할 토픽 목록
            "reason": "string"      # 제안 이유 (채팅창에 노출)
        }
    },
    {
        "name": "apply_topic_filter",
        "description": "사용자 확인 후에만 호출. 토픽 필터를 좌측 패널에 실제 적용",
        "input_schema": {
            "topics": ["string"]
        }
    },
    {
        "name": "change_sort",
        "description": "정렬 순서 변경",
        "input_schema": {
            "sort_by": "유사도순 | 출원일순 | 출원인순"
        }
    },
    {
        "name": "highlight_patent",
        "description": "특정 특허를 좌측 목록에서 강조 표시",
        "input_schema": {
            "patent_id": "string"
        }
    }
]
```

## 4. 대화 흐름 — 확인 후 패널 제어

```
사용자: "이 결과에서 센서 관련만 보고 싶어"
         │
         ▼
  [LLM 내부 — isMeta: true, UI 미노출]
  get_panel_state() → 현재 필터 없음, 결과 50건
  토픽 클러스터에서 "센서·인식" 군집 탐지
         │
         ▼
  [채팅창 AI 응답 — CTA 포함, UI 노출]
  ┌──────────────────────────────────────┐
  │ 🤖 결과 50건 중 센서·인식 관련 토픽   │
  │ 2개를 필터로 걸면 18건으로 좁혀져요.  │
  │                                      │
  │  · 센서 융합 (LiDAR·레이더)  11건    │
  │  · 카메라 기반 인식           7건    │
  │                                      │
  │  [취소]   [이 필터 적용하기 →]       │
  └──────────────────────────────────────┘
         │
    사용자: [이 필터 적용하기 →] 클릭
         │
         ▼
  apply_topic_filter(["센서 융합", "카메라 기반 인식"])
  → 공유 상태 Store dispatch
  → 좌측 패널 즉시 갱신 (18건 표시)
         │
         ▼
  [채팅창 완료 메시지]
  "필터를 적용했어요. 18건이 표시됩니다."
```

## 5. 컨텍스트 주입 구조

매 요청마다 시스템 프롬프트에 좌측 패널 상태를 주입한다.

```python
system_prompt = f"""
당신은 PatSol 특허 검색 어시스턴트입니다.

[현재 세션 컨텍스트]
- 검색 의도: {session.search_intent}
- 적용된 필터: {state.applied_filters}
- 정렬: {state.sort_order}
- 결과 건수: {len(state.results)}건
- 결과 상위 5건: {state.results[:5]}

사용자 요청 유형별 처리:
- 필터 조정 요청  → propose_topic_filter 호출
- 정렬 변경 요청  → change_sort 호출
- 결과 관련 질문  → get_panel_state 후 답변
- 특허 내용 질문  → 직접 답변 (tool 불필요)
"""
```

## 6. Claude.ai isMeta 패턴 적용

[[ia_spec-v0.1]] Reference 섹션의 isMeta 패턴을 채팅-워크플로우 연동에 적용한다.

| 처리 단계 | isMeta | UI 표현 |
|---|---|---|
| get_panel_state 호출 | true | 미노출 (내부 처리) |
| 토픽 클러스터 분석 | true | "분석 중..." 인디케이터만 |
| propose_topic_filter | false | CTA 포함 메시지 완전 노출 |
| apply_topic_filter | true | 좌측 패널 갱신 + 완료 메시지만 |

## 7. 구현 단계 로드맵

| 단계 | 내용 | 선결 조건 | 난이도 |
|---|---|---|---|
| **1단계** | 공유 상태 Store 구축 (Zustand 등). 채팅이 패널 상태 읽기만 가능 | 없음 | 낮음 |
| **2단계** | AI 응답에 CTA 버튼 포함. 클릭 시 Store dispatch로 패널 갱신 | 1단계 | 중간 |
| **3단계** | Tool-use 연동. AI가 자율적으로 tool 호출 결정 | 2단계 | 높음 |
| **4단계** | 다중 tool 연쇄. 필터 적용 → 결과 재요약 → 추가 제안 자동화 | 3단계 | 높음 |

> 1·2단계만으로도 "AI 제안 → 사용자 확인 → 패널 제어" 핵심 UX 구현 가능.
> 3단계부터 AI가 자율적으로 tool을 선택하는 에이전트로 고도화.

## 8. 미결정 항목

| 항목 | 내용 |
|---|---|
| Store 기술 스택 | Zustand / Redux / React Context 중 선택 |
| Tool 실행 권한 범위 | AI가 제안 없이 바로 실행 가능한 액션 범위 정의 (현재: 전부 확인 필요) |
| 다중 액션 처리 | 필터 + 정렬을 동시에 제안할 때 CTA 디자인 |
| 패널 상태 전달 토큰 최적화 | 결과 50건 전체를 컨텍스트에 넣으면 토큰 과다 → 요약 전략 필요 |
