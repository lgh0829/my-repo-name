## 확정된 핵심 결정

### UX 패러다임
- **워크플로우 패러다임** 채택 (AI-first 아님)
- 채팅은 보조 역할 — 검색 결과가 메인, 챗봇은 우측 사이드바
- 챗봇 컨텍스트는 특허 검색 → 분석 리포트 → 청구항 → 명세서까지 이어짐

### 결과 화면 레이아웃
- 2단 구조: **검색결과(좌, 메인)** · **사이드바(우)**
- 사이드바 구성: 분석 대상 문서 패널(상단) + 채팅 패널(하단)

### 검색 경로
| 경로 | 진입 조건 | 검색 엔진 |
|---|---|---|
| 검색식 | 검색식 직접 입력 | Elastic Search |
| 경로 A | 자연어 충분 | Vector DB Top 50 |
| 경로 B-1 | 자연어 불충분 → 토픽 선택 → 예상 50건 이하 | Vector DB Top 50 |
| 경로 B-2 | 자연어 불충분 → 토픽 선택 → 예상 50건 초과 | Vector DB Top 50 |

### OQ 결정 현황 (전부 완료)

| OQ | 결정 내용 |
|---|---|
| OQ-1 | 문서 업로드 후 검색창 자동 채움 (현행 서비스) |
| OQ-2 | 검색식 대분류/소분류 드롭다운 포함 |
| OQ-3 | 채팅 컨텍스트: 현재 세션만 |
| OQ-3a | AI가 후속 질문 의도 구분 → 검색 범위 조정형이면 CTA 제시 → 사용자 확인 후 결과 재진입 |
| OQ-4 | 정렬 옵션 포함, 유사도순 디폴트 |
| OQ-5 | 페이지네이션 (단위 설정 가능) |
| OQ-6 | 분석 리포트 내보내기: PDF |
