# Service Policy Register — PatSol 서비스 정책 등록부

> 이 파일은 PatSol의 UX·제품 의사결정의 근거가 되는 정책을 ID로 관리합니다.
> FRD·스펙 문서에서 `(P-XXX)` 형태로 참조하세요.
> 정책 충돌 발견 시, 설계를 진행하기 전에 먼저 정책을 결정합니다.

---

## P-100 포지셔닝 정책

| ID | 정책명 | 내용 |
|---|---|---|
| P-101 | 외부 검색 수용 | PatSol은 외부 DB(KIPRIS, Google Patents, USPTO 등)에서 가져온 검색 결과를 그대로 Import할 수 있다. PatSol이 검색의 독점적 진입점일 필요는 없다. |
| P-102 | 핵심 가치: 연결 | PatSol의 핵심 가치는 흩어진 선행기술 조사 결과 → 청구항 설계 → 명세서 초안을 하나의 흐름으로 이어주는 것이다. 검색 자체의 품질이 경쟁 우위가 아니다. |
| P-103 | 변리사 주체 원칙 | 변리사가 모든 의사결정의 주체이며, PatSol은 보조 도구다. UX 전반에서 이 관계가 시각적으로도 명확하게 드러나야 한다. |

## P-200 AI 사용 정책

| ID | 정책명 | 내용 |
|---|---|---|
| P-201 | AI 대체 인상 금지 | "AI가 대체한다"는 인상을 주는 표현·디자인 패턴을 사용하지 않는다. 변리사의 전문 판단이 최종 결정임을 시각·카피 전반에서 일관되게 유지한다. |
| P-202 | 명세서 AI 전면화 금지 | 명세서 작성 영역에서 AI를 전면에 내세우지 않는다. "AI 명세서 작성", "자동 생성" 등의 표현은 변리사의 전문직 정체성 위협으로 받아들여질 수 있다. |
| P-203 | AI 보조 표현 권장 | AI 역할을 표현할 때는 `초안 작성 지원`, `작성 효율화`, `변리사를 돕는` 등 변리사가 주체임을 유지하는 표현을 사용한다. |
| P-204 | 검색 영역 AI 수용 | 검색 영역에서는 AI를 "편리한 도구"로 전면에 드러내도 무방하다. 변리사의 검색 작업을 AI가 보조하는 것은 전문직 위협으로 인식되지 않는다. |

## P-300 검색 정책

| ID | 정책명 | 내용 |
|---|---|---|
| P-301 | Top-N 결과 제한 | 기본 검색 결과는 Top 50건 이내로 제한한다. 변리사 실무에서 50건 초과 검토는 비효율적이라는 가설(H3)에 근거. |
| P-302 | 전수 수집 옵션 제공 | 사용자가 명시적으로 요청할 경우 Top-N 제한 없이 전수 수집 모드를 제공한다. |
| P-303 | 토픽 필터 선택 방식 | 토픽 카드 미선택 시 해당 토픽의 결과가 완전히 제외되지 않는다. 순위가 낮아질 뿐이다. |
| P-304 | 다국어 검색 지원 범위 | `[미결]` 한국어 검색식 기반 검색을 우선 지원한다. 영어·일본어 자동 번역 검색 지원 범위는 결정 필요. |

## P-400 데이터 및 저장 정책

| ID | 정책명 | 내용 |
|---|---|---|
| P-401 | 검색 이력 보존 | 검색 이력은 워크스페이스 내에서 영구 보존되며, 사용자가 명시적으로 삭제하기 전까지 유지된다. |
| P-402 | 사무소 스타일 학습 | `[미결]` 변리사 또는 사무소별 명세서 작성 스타일을 학습·반영하는 기능의 데이터 소유권과 공유 범위 결정 필요. |
| P-403 | 출원 전 기밀 보호 | 출원 전 발명 내용은 외부에 노출되어서는 안 된다. 데이터 처리 방식(온프레미스 옵션 등)은 별도 결정 필요. |
