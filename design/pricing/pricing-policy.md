## 2. 플랜별 기능 제공량

> ※ 제공량은 **월(30일) 기준**이며, 연간 구독 시 매월 순차 지급됩니다.
> ※ 검색(`search`)은 전 플랜 무제한이나, **서버 과부하 방지를 위해 시간당 60회로 제한**됩니다.

| 기능 | Starter | Basic | Pro | Search |
|---|---|---|---|---|
| **검색** (`search`) | 무제한 (시간당 60회) | 무제한 (시간당 60회) | 무제한 (시간당 60회) | 무제한 (시간당 60회) |
| **명세서 초안 생성** (`write`) | 2회/월 | 10회/월 | 20회/월 | - |
| **회피 분석 시 선택 건수** | 3건/월 | 3건/월 | 5건/월 | - |

## 3. 구독권 지급 방식

### 3-1. 구독 시작일(기준일) 설정

결제 시점의 기존 구독 상태에 따라 시작일이 다르게 적용됩니다.

| 케이스 | 조건 | 시작일 |
|---|---|---|
| **기간 연장** | 동일 플랜 + 아직 만료 전 | 기존 `expire_date` (이어서 연장) |
| **신규 / 플랜 변경** | 구독 만료 후 재구독 또는 다른 플랜으로 변경 | 결제일 (오늘) |

### 3-2. 만료일(`expire_date`) 연장

산정된 시작일 기준으로 아래와 같이 연장되며, **4개의 하위 만료일 필드(검색·명세서·분석·`expire_date`)가 동일한 날짜로 동기화**됩니다.

| 구독 유형 | 연장 기간 |
|---|---|
| 월간 구독 | 시작일 + 30일 |
| 연간 구독 | 시작일 + 365일 |

### 3-3. 기능별 사용량 지급 방식

**① 명세서 작성권 (`write`) — 배열(Array)로 관리**

- 단순 숫자 누적이 아닌, **발급일·사용기한이 명시된 객체(Object)** 형태로 `extra['write']` 배열에 추가
- 월간 구독: 한 달 치 사용기한을 가진 객체 **1개** 추가
- 연간 구독: 30일 단위로 쪼개어 **12개 객체를 한 번에** 배열에 추가 (매월 순차 소진)

```javascript
// 연간 구독 시 extra['write'] 예시
[
  { issued_at: "2025-07-01", expires_at: "2025-07-31", count: 20 },
  { issued_at: "2025-07-01", expires_at: "2025-08-31", count: 20 },
  ...  // 총 12개
]
```

**② 검색(`search`) · 청구항(`claim`) — 정수(Integer)로 관리 (현재 미사용)**

- 단순 정수형으로 관리하며, **기존 잔여 횟수 + 이번 플랜 제공 횟수**로 합산하여 덮어씁니다.
- `extra['search']`, `extra['claim']` 필드에 반영

> 현재 **검색(`search`) · 청구항(`claim`) 사용량을 따로 지급하지 않습니다. (무제한)**

## 4. 환불 정책

> 환불 문의: contact@patsol.kr / 070-8028-3925 (평일 09:00–18:00)

### 4-1. 청약 철회 (7일 이내 단순 변심)

- 유료서비스 공급일로부터 **7일 이내**, 명세서 작성(`write`) 기능을 **사용하지 않은 경우** 전액 환불 가능
- 철회 의사는 서비스 홈페이지를 통해 표시하며, 회사 수령 즉시 효력 발생

### 4-2. 7일 경과 후 환불

전액 환불 불가. 아래 공식에 따라 환불 금액 산정:

```
환불금액 = 전체 이용대금
           - MAX(기 이용일수 해당금액, 이용회차(명세서작성) 해당금액)
           - 잔여 이용요금의 10%
```

- 연간 결제 후 단기(월간) 이용 기간이 경과한 시점에 해지 시 → **단기 할인율(월간 단가) 기준으로 재산정**

### 4-3. DB 처리 기준 (환불 시 안전한 롤백 방법)

> 아래는 **위험한 방식(즉시 초기화)을 반드시 피하고**, 안전한 방식으로만 처리해야 합니다.

**① `expire_date` 처리**

| 방식 | 내용 |
|---|---|
| 위험 | `now()`로 즉시 덮어쓰기 → 이전 결제 잔여일 및 관리자 연장 기간 소멸 |
| 안전 | 이번 결제로 부여된 기간만큼만 차감 (월간: -30일 / 연간: -365일). 단, 결과가 오늘보다 과거라면 `now()`로 보정 |

**② 기능 잔여 횟수 처리**

| 항목 | 방식 |
|---|---|
| 위험 | 일괄 `0` 초기화 → 이벤트 지급분·마이그레이션 이력 소멸 |
| `search` · `claim` (정수형) | 이번 요금제 기본 제공량만큼만 차감. 결과가 음수면 `0`으로 보정 |
| `write` (배열형) | 이번 결제건으로 발급된 객체만 핀셋 삭제 |

**③ 환불 처리 방법**

- 환불은 **Portone에서 수동 처리**
- 환불 확인 후 **3영업일 이내** 동일 결제수단으로 환급 원칙
- 부대비용·수수료는 차감 후 환불 가능

### 4-4. 어뷰징 방지 기준

아래 조건 중 **하나라도 해당하면 전액 환불 불가**:

| 조건 | 기준 |
|---|---|
| 이용일수 초과 | 결제일로부터 **7일 경과** |
| 기능 사용 | 명세서 작성(`write`) **1회 이상 사용** |

> ※ 추가 기준(사용률 N% 이상 시 부분 환불 제한 등)은 추후 기획 확정 후 본 문서에 반영 예정 (TBD)

### 4-5. 기타 환불 불가 사유

- 유료서비스 이용 중 **회원 탈퇴** 시 즉시 해지, 환불 불가
- **선물·프로모션·무상 취득** 유료서비스는 환불 대상 아님
- 이용요금 **체납 시** 자동 이용권 해지 (정기결제)
- 회원 약관 위반으로 인한 강제 해지의 경우 손해배상액 공제 후 환불
