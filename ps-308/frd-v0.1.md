## 1. 개요

### 1.1 목적

변리사가 선행기술 조사를 위해 특허 문서를 검색할 때, AI가 사용자 의도를 파악하여 Top 50건 이내에 원하는 문서가 모두 포함되도록 검색 정밀도를 높인다.

### 1.2 사용자

| 유형 | 설명 |
|---|---|
| 주 사용자 | 변리사, 특허 담당자 |
| 검색 목적 A | 특정 발명의 선행기술 파악 (정밀 검색) |
| 검색 목적 B | 기술 영역 전수 조사 (광범위 수집) |

### 1.3 핵심 가설

| # | 가설 | 검증 지표 |
|---|---|---|
| H1 | 사용자는 자연어로 검색 의도를 충분히 표현할 수 있다 | AI 충분도 판정 통과율 |
| H2 | AI 토픽 카드가 사용자 의도를 정확히 반영한다 | 토픽 카드 선택률 / 재검색률 |
| H3 | Top 50건이 변리사 실무에 충분하다 | "더 보기" 요청 빈도 / 전수 수집 선택 비율 |

> **⚠ 우선 검증 가설:** H2가 가장 위험한 가설. AI 토픽이 변리사의 기술 분류 감각과 맞지 않으면 전체 흐름이 무너짐. MVP 초기에 토픽 카드 선택률을 최우선 지표로 모니터링할 것.

## 2. 페이지 구조

```
/patent (로그인 후)
├── 검색 목적 선택 (정밀 검색 / 전수 수집)
├── 문서 Import 영역
├── 자연어 검색 입력
├── 검색 실행 흐름
│   ├── 경로 A: 직접 검색 (충분한 입력)
│   └── 경로 B: 심층 검색 (불충분한 입력)
│       ├── 1단계: 토픽 Q&A
│       └── 2단계: 필터 / 검색식 보강
├── Breadcrumb 탐색 바 (sticky)
└── 검색 결과 섹션 (공통 출력)
```

## 3. 기능 상세

### 3.1 검색 목적 선택

**위치:** 페이지 진입 시 검색 입력 전 선택

| 목적 | 설명 |
|---|---|
| 정밀 검색 | 원하는 특허를 Top 50 이내로 좁혀서 찾기 |
| 전수 수집 | 관련 특허를 빠짐없이 모두 수집 (50건 초과 가능) |

**목적별 동작 차이**

| | 정밀 검색 | 전수 수집 |
|---|---|---|
| AI 충분도 판단 | 적용 | 스킵 |
| 결과 건수 제한 | Top 50 고정 | 제한 없음 |
| 후속 UX | 토픽 카드 → 필터로 좁히기 | 전체 결과 반환 + 일괄 내보내기 |

> **※ OQ-1:** 전수 수집 진행 전 확인 단계 문구 및 방식 미결정 — "현재 조건으로 N건이 검색됩니다. 계속하시겠습니까?" 모달 vs 인라인 안내

---

### 3.2 문서 Import

**위치:** 자연어 검색 입력 영역 상단

| # | 문서 유형 | AI 처리 내용 |
|---|---|---|
| 1 | 직무발명신고서 | 텍스트 및 이미지 추출 후 검색에 적합한 형태로 변환 |
| 2 | 도면 | 동일 |
| 3 | 클라이언트 회의록 | 동일 |
| 4 | 기타 발명문서 | 동일 |

**지원 파일 형식:** `.pdf` `.doc` `.docx` `.ppt` `.pptx` `.hwp` `.hwpx`

> **※ OQ-2:** 업로드 후 AI 추출 텍스트 처리 방식 미결정
> - 옵션 A: 검색창에 자동 채움
> - 옵션 B: 별도 미리보기 패널 표시 후 사용자 확인

---

### 3.3 자연어 검색 입력

| 요소 | 설명 |
|---|---|
| 입력 가이드 | "기술 분야, 해결 문제, 핵심 구성을 포함해서 입력하세요" |
| 검색창 | 멀티라인 자유 서술 입력 |
| 검색 버튼 | 입력 제출 → AI 충분도 판단 트리거 |
| 입력 초기화 | 검색창 내용 전체 삭제 |

---

### 3.4 검색 실행 흐름

#### AI 충분도 판단 기준

| 판정 | 기준 | 진입 경로 |
|---|---|---|
| 충분 | 기술 키워드 2개 이상 + 목적/문제 맥락 포함 | 경로 A |
| 불충분 | 단어 1~2개, 맥락 없음 | 경로 B |

- **충분 예시:** "리튬이온 배터리 급속충전 발열 억제 열관리 BMS 제어"
- **불충분 예시:** "배터리", "좋은 기술 찾아줘"

---

#### 경로 A — 직접 검색

**진입 조건:** AI 충분도 판정 "충분"

| 단계 | 내용 |
|---|---|
| 1 | Vector DB 검색 즉시 실행 (Top 50) |
| 2 | 검색 결과 섹션으로 이동 |

---

#### 경로 B — 심층 검색 (2단계 깔때기)

**진입 조건:** AI 충분도 판정 "불충분"

**1단계: 토픽 Q&A**

| 요소 | 설명 |
|---|---|
| 토픽 카드 | Vector DB 클러스터링으로 상위 토픽 자동 생성, 카드 형태로 제시 |
| 선택 방식 | 토픽 카드 선택 또는 직접 텍스트 입력 |
| 예상 건수 표시 | 선택 토픽 기준 예상 검색 결과 건수 실시간 표시 |

**1단계 후 분기**

| 조건 | 다음 단계 |
|---|---|
| 예상 건수 50건 이하 | 검색 즉시 실행 → 결과 섹션 이동 |
| 예상 건수 50건 초과 | 2단계 (필터 / 검색식 보강) 진입 |

**2단계: 필터 / 검색식 보강**

| 요소 | 설명 |
|---|---|
| AI 추천 필터 | 출원일, 출원인, IPC 코드 등 적합한 필터 항목 자동 추천 |
| 직접 검색식 입력 | Boolean, CPC 코드 등 전문 검색식 직접 입력 옵션 |
| 검색 실행 | 조건 확정 후 Vector DB 검색 (Top 50) → 결과 섹션 이동 |

> **※ OQ-3:** 검색식 직접 입력의 백엔드 처리 방식 미결정 — Elastic Search 연동 vs Vector DB 필터링

### 3.5 Breadcrumb 탐색 바

**위치:** GNB 바로 아래 sticky 고정. 검색 결과 진입 이후부터 표시.

**구조 예시**

```
① 검색  ›  ② 토픽 선택  ›  ③ 필터  ›  ④ 결과
"배터리"    열관리시스템×    2020년~×    ← 현재
```

**단계별 상태**

| 상태 | 스타일 | 동작 |
|---|---|---|
| 현재 단계 | 굵게 + 밑줄 | 클릭 불가 |
| 이전 단계 | 링크 스타일 | 클릭 시 해당 단계로 복귀 |
| 미래 단계 (미진입) | 회색, 비활성 | 클릭 불가 |

**각 단계의 표시값**

| 단계 | 표시 내용 |
|---|---|
| ① 검색 | 입력한 자연어 검색어 (20자 초과 시 말줄임 처리) |
| ② 토픽 선택 | 선택한 토픽명 + 삭제(×) 버튼 |
| ③ 필터 | 적용된 필터 조건 요약 + 삭제(×) 버튼 |
| ④ 결과 | "결과 N건" |

**이전 단계 복귀 동작**

| 복귀 위치 | 표시되는 UI | 이후 단계 처리 |
|---|---|---|
| ① 검색 클릭 | 토픽 카드 재표시 (재선택 가능) | ②③ 비활성 유지, 값 보존 |
| ② 토픽 선택 클릭 | 필터 패널 재표시 (재설정 가능) | ③ 비활성 유지, 값 보존 |

> **※ OQ-4:** 복귀 후 이후 단계의 값(토픽, 필터) 초기화 vs 보존 정책 결정 필요

---

### 3.6 검색 결과 섹션

**진입 경로:** 경로 A, 경로 B 1단계, 경로 B 2단계 모두 동일한 결과 화면으로 합류

**상단 영역**

| 요소 | 설명 |
|---|---|
| 결과 건수 | "N건" 표시 |
| 진입 경로 레이블 | 경로 A / 경로 B / 전수 수집 중 하나 표시 |
| 적용된 조건 칩 | 토픽, 필터 조건 칩 나열. 각 칩에 삭제(×) 버튼 |
| 다시 검색 버튼 | 검색 입력 단계로 복귀 |

**AI 토픽 카드**

| 요소 | 설명 |
|---|---|
| 카드 구성 | 토픽명 + 해당 건수 |
| 최대 카드 수 | 4개 |
| 선택 동작 | 클릭 시 해당 토픽으로 결과 필터링 |

**특허 결과 카드**

| 요소 | 설명 |
|---|---|
| 발명 명칭 | 특허 제목 |
| 메타 정보 | 출원번호, 출원인, 출원일, IPC 코드 |
| 유사도 | Vector DB 유사도 점수 (%) |
| 요약 발췌 | 발명의 설명 일부 발췌 |
| 태그 | 주요 기술 키워드 |

> **※ OQ-5:** 결과 표시 방식 미결정 — 페이지네이션 vs 무한스크롤

## 5. 비기능 요구사항

| 항목 | 기준 |
|---|---|
| AI 충분도 판단 응답 시간 | 입력 완료 후 1초 이내 |
| Vector DB 검색 응답 시간 | 3초 이내 |
| 토픽 카드 생성 응답 시간 | 5초 이내 |
| 지원 파일 업로드 크기 | 최대 50MB (파일당) |
