### 2.2.2 자연어 검색

| No. | D3 기능/컴포넌트 | D4 세부항목 | 기능설명/정책 | 구현여부 |
|---|---|---|---|---|
| 2.2.2.1 | 자연어 검색 | 자연어 검색 입력 가이드 | 발명 내용을 자유롭게 서술하거나 발명신고서·청구항 문장을 그대로 붙여넣기 할 수 있도록 안내. placeholder 예시로 청구항 형식 문장 제공 | Y |
| 2.2.2.2 | 자연어 검색 | 특허 문서 검색 충분도 파악 | Vector DB 검색 결과 Top 50 이내에 원하는 문서를 담을 수 있을 만큼 입력이 구체적인지 AI가 판단. 충분/불충분 분기 결정. 충분 예시: "리튬이온 배터리 급속충전 시 발열을 억제하기 위한 열관리 알고리즘 및 BMS 제어 방법" / 불충분 예시: "배터리", "좋은 기술 찾아줘" | Y |
| 2.2.2.3 | 자연어 검색 | 직접 검색 실행 | 충분한 입력을 받았을 때 VectorDB로 특허 문서를 검색 후(Top 50) 검색 결과 섹션으로 이동 | - |
| 2.2.2.4 | 자연어 검색 | 심층 검색 실행 | 충분치 않은 입력 시 VectorDB에서 입력 기반 클러스터링으로 상위 토픽 생성 | - |
| 2.2.2.5 | 자연어 검색 | 상위 토픽 Q&A 폼 생성 | 상위 토픽을 카드 형태로 제시. 사용자가 선택하거나 직접 입력해서 제출 가능. "선택 없이 그냥 검색" 옵션 제공 | - |
| 2.2.2.6 | 자연어 검색 | Q&A 폼 입력 기반으로 검색 실행 | 사용자의 의도가 충분히 구체화되었다면 VectorDB로 특허 문서를 검색 후(Top 50) 검색 결과 섹션으로 이동 | - |
| 2.2.2.7 | 자연어 검색 | 필터 / 검색식 보강 (2단계) | 토픽 선택 후에도 예상 건수가 50건 초과 시 출원일·출원인·IPC 코드 필터 또는 직접 검색식 입력으로 추가 좁히기 | - |

## 검색 실행 흐름 요약

```
검색 입력
├── 상세 검색식 입력
│   └── 문법 오류 감지 → 수정 제안(사용자 승인)
│       └── Elastic Search → 2.3 검색 결과
└── 자연어 입력
    └── AI 충분도 판단
        ├── 충분 → Vector DB (Top 50) → 2.3 검색 결과          [경로 A]
        └── 불충분
            └── 클러스터링 → 상위 토픽 Q&A 폼
                ├── 50건 이하 → Vector DB (Top 50) → 2.3 검색 결과   [경로 B-1]
                └── 50건 초과 → 필터/검색식 보강 → 2.3 검색 결과      [경로 B-2]

2.3 검색 결과
├── 채팅 (새 주제 / 이어서 질문)
├── 검색 결과 조회 (필터링 / 정렬 / 상세 조회)
└── 분석 대상 문서 담기 (추가 / 제외 / 분석 리포트 / 상세 조회)
```
