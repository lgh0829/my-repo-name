프롬프트 구현은 사무소명보다 클러스터와 대리인 매핑을 우선한다. CPC는 보조 지표다.

## 클러스터

| 클러스터 | 이름 | 핵심 |
|---|---|---|
| C1 | JP 번역체 | 실시형태 + 선행기술 특허문헌 인용 + 간접 과제 |
| C3 | JP 무효과형 | 실시형태 + 효과 섹션 없음 |
| C2 | 국내 표준 | 실시예 + 효과 기재 + 실시형태 미사용 |
| C4 | 배경기술 중심 | 긴 배경기술 + 짧은 실시예 + 직접 과제 |
| C0 | 장문 간접형 | 간접 과제 + 긴 실시예 |

## 프롬프트 파라미터

| 파라미터 | C1/C3 | C2 | C4 | C0 |
|---|---|---|---|---|
| 실시 용어 | 실시형태 | 실시예 | 실시예 | 혼재 |
| 선행기술 인용 | 강함 | 약함 | 중간 | 중간 |
| 과제 어법 | 간접 | 직접 | 직접 강함 | 간접 |
| 실시예 구조 | 복수 가능 | 단일 중심 | 단일 짧음 | 장문 |
| 효과 섹션 | C1 기재 / C3 생략 | 기재 | 기재 | 기재 |
| 배경기술 | 짧음 | 표준 | 긺 | 표준 |
| 실시예 | 긺 | 중간 | 짧음 | 긺 |

## 운영 규칙

- 대리인을 알면 클러스터 추정에 우선 사용한다.
- CPC만으로 클러스터를 확정하지 않는다.
- C2는 leeandmok형과 김앤장형이 섞이므로 용어정의 관용구 여부와 Embodiment 분량으로 하위 구분한다.
