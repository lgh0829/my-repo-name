# Embodiment Analysis Harness

`발명을 실시하기 위한 구체적인 내용` 전문 또는 발췌를 분석할 때 사용한다.

## 입력 처리

1. ES를 1회 짧게 접속 시도한다.
2. 출원번호가 있으면 ES에서 `Embodiment`와 메타데이터를 조회한다.
3. ES 접속 실패 또는 빈 결과면 사용자가 준 텍스트로 분석한다.

출력 첫 줄에 출처를 표시한다.

```text
[출처: ES 조회 / 출원번호 ...]
[출처: 붙여넣은 텍스트(ES 미접속)]
```

## 분석 차원

| 차원 | 볼 것 |
|---|---|
| 도입 정형구 유형 | JP 번역체 / 국내 표준 / 권리범위 면책형 |
| 도입부 선행 요소 | 도면 목록 / 용어 처리 / 권리범위 면책 / 바로 구성설명 |
| 실시 용어 | `실시예` / `실시형태` / `구현예` |
| 실시예 개수 | 단일 실시예 vs 복수 실시예(`제1`, `제2` 마커) |
| 도면 연계 밀도 | 참조번호 빈도, 도면당 설명 분량 |
| 변형예·응용예 | 실시예만 있는지, 변형예/응용예 포함인지 |
| 수치한정 표현 | 구체적 수치 범위 vs 추상적 기술 |
| 용어 처리 유형 | 실질 정의 / 보일러플레이트 / 서수사 관용구 / 인라인 약칭 |
| 도면 참조 방식 | 괄호형 `(10)` / 인라인형 `도 1` / 혼용 |
| 전개 순서 | 도입 → 선행요소 → 구성 → 동작 → 변형예 |
| 분량·밀도 | 글자 수, 참조 빈도, 발췌 여부 |

## 용어 처리 4유형

| 유형 | 의미 | 예 |
|---|---|---|
| 실질 용어 정의 | 발명 고유 용어를 정의 | `"X"란 ...를 의미한다` |
| 해석 보일러플레이트 | 해석 규칙 상투구 | `포함`의 의미, 단수=복수, 용어 설명 선언 |
| 서수사 관용구 | 제1·제2는 순서가 아니라 구별 목적 | "제1, 제2 등의 용어는..." |
| 인라인 약칭 | 본문 중 괄호 약칭 | `(이하 "..."라 함)` |

주의: "용어를 설명하겠다"는 선언이나 "포함/단수의 해석규칙"은 실질 용어 정의가 아니다.

## 출력 형식

```markdown
## 전개방식 분석: {출원번호 또는 입력 텍스트}

### 메타
- 발명명칭:
- 출원인:
- CPC:
- 대리인:

### 8개 차원
| 차원 | 판정 | 근거 |
|---|---|---|

### 클러스터 추정
- 추정 클러스터:
- 근거:

### 한줄 요약
```
