# Report and Prompt Format Harness

## 산출물 위치

- 리포트: `output/*.md`
- 데이터: `output/*.csv`
- 앱: 저장소 루트 또는 전용 앱 디렉터리
- DB 생성 스크립트: `scripts/db/`
- 로컬 DB/빌드 로그: `data/local/`

## 공통 출력 규칙

- 수치 테이블은 markdown 테이블로 작성한다.
- 쿼리와 프롬프트는 코드 펜스로 감싼다.
- 데이터 기반 사실과 해석을 분리한다.
- 개인정보(발명자명·주소)는 포함하지 않는다.
- 같은 파일을 갱신할 때는 기존 결론과 충돌하는 구수치를 반드시 정정한다.

## 리포트 기본 구조

```markdown
# [분석 대상] 명세서 스타일 분석 리포트

## 요약
- 분석 기간:
- 분석 건수:
- JP 포함/제외:
- 핵심 스타일 특성:

## 데이터 개요
- 수집 쿼리 조건
- 유효 문서 수 / 제외 문서 수 및 사유

## 스타일 분석 결과
### 청구항 구조
### 독립항 전략
### 표현 방식
### 도면 참조 방식
### 명세서 섹션 구성
### Embodiment 전개 방식
### 본문 서술 패턴
### CPC / 대리인 분포

## 인사이트
## 스타일 프롬프트
## 부록
```

## 스타일 프롬프트 기본 구조

```markdown
### 적용 조건
- 사무소/대리인:
- CPC/발명 본질 유형:
- JP 포함/제외:

### 청구항 작성 규칙
- 독립항:
- 종속항:
- 전략 패턴:

### 본문 작성 규칙
- 배경기술:
- 해결하려는 과제:
- 과제 해결 수단:
- 발명의 효과:
- 실시예:

### 표현 방식
- 선호 표현:
- 피해야 할 표현:

### 도면 참조
- 방식:

### 금지 사항
```
