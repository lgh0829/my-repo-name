# 동작-시간형 판정 기준 검증 1단계

작성일: 2026-06-19  
데이터 소스: `data/local/patents_local_v2.db`  
분석 범위: 청구항이 존재하는 v2 로컬 DB 문서 2,459건

## 1. 결론 요약

동작-시간형은 CPC나 도면 키워드가 아니라 **독립항 구성에서 발명의 본질이 시간 흐름에 따른 동작방법인지**로 먼저 판정해야 한다.

| 항목 | 결과 |
|---|---:|
| 청구항 존재 문서 | 2,459건 |
| 청구항 기준 동작-시간형 | 269건 (10.9%) |
| 동작-시간형 중 CPC `G06/G11/G16` 포함 | 143건 (53.2%) |
| 동작-시간형 중 도면 키워드 `흐름도/순서도/플로우차트/블록도` 포함 | 230건 (85.5%) |
| 동작-시간형 중 Embodiment 시간·단계 키워드 포함 | 249건 (92.6%) |
| CPC `G06/G11/G16` 문서 중 청구항 기준 동작-시간형 | 143/398건 (35.9%) |
| 도면 키워드 포함 문서 중 청구항 기준 동작-시간형 | 230/1,022건 (22.5%) |

핵심 해석은 두 가지다.

1. CPC `G06/G11/G16`은 동작-시간형을 잘 설명하는 축이지만, 청구항 기준 동작-시간형의 46.8%는 이 CPC 밖에 있다.
2. 도면의 흐름도/블록도 키워드는 재현율이 높지만(85.5%), 그 키워드가 있다고 모두 동작-시간형은 아니다. 도면 키워드 포함 문서 중 실제 동작-시간형은 22.5%에 그쳤다.

따라서 CPC와 도면 키워드는 **보조 검증 변수**로 쓰고, 1차 판정은 독립항에서 해야 한다.

## 2. 청구항 기반 판정식

이번 검증에서는 다음 프록시를 사용했다.

| 단계 | 기준 |
|---|---|
| 독립항 추출 | 청구항을 `¶` 기준으로 나눈 뒤, 앞부분에 `제n항에 있어서`, `제n항에 따른`, `제n항의` 의존 표현이 있는 항을 종속항으로 제외 |
| 방법항 후보 | 독립항에 `동작/제어/처리/관리/생성/전송/수신/저장/검사/검출/보정/예측/제공/학습/인식/판단/결정/스케줄링/구동 방법` 또는 `방법으로서` 포함 |
| 시간 흐름성 | 방법항에 `단계`, `S100`류 단계가 2회 이상 있고, 시간·순서 표현 또는 주체 기반 수행 흐름이 존재 |
| 제외 | `제조/시공/추출/증착/형성/코팅/합성/배양/치료 방법`은 공정·제조형으로 제외 |

주의: 이 단계는 최종 법적 판단이 아니라 정량 검증용 프록시다. 특히 기판 처리, 반도체 장비 제어처럼 공정형과 동작형이 섞이는 영역은 후속 수동 검토가 필요하다.

## 8. 다음 단계 제안

이번 단계에서는 “동작-시간형인지 여부”를 청구항 기반으로 먼저 검증했다. 다음 단계의 Embodiment 전개 구조 분석은 이 269건을 모집단으로 삼아야 한다.

후속 분석 축은 다음이 적절하다.

| 축 | 볼 것 |
|---|---|
| 블록도 선행 여부 | 장치/시스템 구성 설명 후 방법 흐름으로 넘어가는지 |
| S단계 밀도 | `S100`, `S200` 등 단계 번호가 본문 전개를 지배하는지 |
| 주체-동작 반복 | `컨트롤러가`, `프로세서가`, `서버가` 등 주체별 수행 문장 반복 여부 |
| 시간 조건 | 클럭, 주기, 프레임, 슬롯, 지연, 응답 조건 중심인지 |
| 변형예 위치 | 기본 흐름 설명 후 예외·변형 조건을 붙이는지 |

현재 결론상 CPC `G06/G11/G16` 또는 도면 키워드만으로 모집단을 만들면 누락과 오탐이 모두 크다. 따라서 후속 Embodiment 전개 분석은 반드시 청구항 기반 모집단에서 시작해야 한다.
