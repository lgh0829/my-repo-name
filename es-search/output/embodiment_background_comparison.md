## 1. 섹션 구조 (1-4)

### 1-1. 공통점

두 사무소 모두 **TechnicalField → Background → Embodiment 3단 구조**만 사용한다.

| 섹션 | jang | leeandmok |
|---|:---:|:---:|
| TechnicalField | 1,000/1,000건 (100%) | 999/1,000건 (99.9%) |
| Background | 1,000/1,000건 (100%) | 1,000/1,000건 (100%) |
| Embodiment | 1,000/1,000건 (100%) | 1,000/1,000건 (100%) |
| TechProblem | **0건** | **0건** |
| TechSolution | **0건** | **0건** |
| AdvantageousEffects | **0건** | **0건** |

TechProblem·TechSolution·AdvantageousEffects는 두 사무소 모두 독립 섹션으로 사용하지 않는다. 해당 내용은 Background 또는 Embodiment 내부에 통합하여 서술하는 것으로 추정된다.

### 1-2. 분량 차이

| 섹션 | jang 평균 | jang 범위 | leeandmok 평균 | leeandmok 범위 |
|---|---:|---|---:|---|
| TechnicalField | **76자** | 18 ~ 864자 | 측정값 없음 (짧고 명확) | — |
| Background | **714자** | 39 ~ 8,388자 | **729자** | 65 ~ 5,353자 |
| Embodiment | **27,069자** | 3,612 ~ 185,653자 | **20,984자** | 2,110 ~ 176,029자 |

**핵심 차이:**

- **Embodiment**: jang이 leeandmok보다 평균 **29% 더 길다** (27,069 vs 20,984자). 최솟값도 jang(3,612자) > leeandmok(2,110자). JP 번역 출원 특성상 원문 일본 명세서의 상세 기재 관행이 반영된 결과로 해석된다.
- **Background**: 평균은 거의 같으나(714 vs 729자) 최댓값 차이가 크다(8,388 vs 5,353자). jang 일부 JP 번역 출원에서 배경기술을 극히 길게 기재하는 사례가 있다.
- **TechnicalField**: jang은 76자로 매우 짧은 1~2문장이 표준. leeandmok도 같은 경향이나 수치 미측정.

## 2. Embodiment 전개 방식 (1-5)

### 2-1. 도입 정형구

| | jang | leeandmok |
|---|---|---|
| **정형구 수** | **3종** (JP 번역체) | **2종** (한국어 표준체) |
| 정형구 ① | "이하, 본 발명의 실시 형태에 대해 도면을 참조하여 설명한다." | "이하에서 예시적인 구현예들에 따른 ~에 대하여 상세히 설명한다." |
| 정형구 ② | "본 발명의 실시형태에 대해서 도면을 사용하여 자세히 설명한다." | "아래에서는 첨부한 도면을 참조하여 본 발명의 바람직한 실시 예에 대하여 상세히 설명한다." |
| 정형구 ③ | "이하, 본 발명에 대하여, 도면을 적절히 참조하면서 상세하게 설명한다." | 없음 |

jang의 3종은 모두 일본 원문 번역체("以下、~について説明する" 계열). leeandmok은 한국어로 작성된 2종만 사용.

### 2-2. 전개 순서

| 단계 | jang | leeandmok |
|:---:|---|---|
| 1 | **도면 목록 소개** ("도 1은 ~의 사시도이다, 도 2는…") | **용어 정의** ("본 명세서에서 '~'란 ~를 의미한다") |
| 2 | 구성 설명 (도면 번호 기반) | 도면 기반 구성 설명 |
| 3 | 동작 원리 | 동작 원리 |
| 4 | 변형 실시예 | 변형 실시예 |

**도입부의 차이가 핵심이다.** jang은 도면 목록을 먼저 나열하는 일본 명세서 관행을 따른다. leeandmok은 용어 정의를 먼저 배치하는데, 이는 삼성전자·삼성디스플레이 계열 명세서에서 청구 범위 해석 분쟁에 대비하는 관행에서 비롯된 것으로 해석된다.

### 2-3. 실시 표현 용어 및 반복 밀도

| | jang | leeandmok |
|---|---|---|
| 주 사용 용어 | **"실시형태"** / "실시 형태" (JP 번역 전용) | **"실시예"** / "실시 예" |
| 언급 횟수 (평균) | **47.3회** / 100건 | **55.4회** / 94건 |
| 출현 비율 | 100% (100/100건) | 94% (94/100건) |

- jang 최신 1,000건 서브샘플에서는 "실시형태"가 100% 출현하지만, 전체 53,925건 기준으로는 "실시예"가 우세(8,468 vs 4,924건). 최신 출원일수록 JP 번역체 "실시형태"가 집중됨을 의미한다.
- leeandmok은 "실시예"를 전 분야에서 일관되게 사용한다.
- 반복 횟수는 leeandmok(55.4회)이 jang(47.3회)보다 많다. jang은 구성·동작을 길게 서술하는 방식, leeandmok은 실시예 번호를 자주 갱신하며 케이스를 나누는 방식의 차이로 해석된다.

## 4. CPC 무관 여부 판정

| 특성 | CPC 무관 여부 | 근거 |
|---|:---:|---|
| TF-BG-EMB 3단 섹션 구조 | ✅ 완전 무관 | 양 사무소 전 CPC에서 예외 없음 |
| TechProblem 등 미사용 | ✅ 완전 무관 | 양 사무소 전 CPC 0건 |
| "실시형태" 용어 (jang) | ✅ 무관 | 전 CPC 63~100% 유지 |
| "실시예" 용어 (leeandmok) | ✅ 무관 | 전 CPC 75~98% 유지 |
| 용어 정의 선행 (leeandmok) | ✅ 무관 | 전 CPC 30~57% 유지 |
| jang EMB > leeandmok EMB 방향성 | ✅ 무관 | 전 CPC에서 jang이 더 긺 |
| Embodiment 절댓값 | ❌ CPC 영향 있음 | jang 내 A(51,014) vs B(19,580): 2.6배 차 |
| 도면 목록 선행 (jang) | ❌ CPC 영향 있음 | C(40.2%) vs G(8.6%): 5배 차 |

**결론**: 어떤 용어를 쓰고 어떻게 시작하는지는 CPC를 막론한 **사무소 고유 특성**이다. 얼마나 길게 쓰는지와 도면 목록 선행 여부는 **기술분야의 영향도 함께 받는다**.
