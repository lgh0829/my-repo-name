## 5. 핵심 상태값 설명

### 원문 위치

도 3 후반 및 도 4~11 진입 전 설명 구간.

### 원문 요지

소거간격 매니저는 메모리 블록의 소거시점을 소거 지시자로 관리한다. 소거 지시자는 설정된 범위 내에서 라운드-로빈 방식으로 갱신될 수 있고, 메모리 블록이 클로즈될 때 해당 메모리 블록의 메타 데이터로 기록된다.

### 분석

동작-시간형 실시예에서 매우 중요한 구간이다. 여기서는 단순 구성요소가 아니라 “시간에 따라 변하는 상태값”을 정의한다.

`소거 지시자(EI)`는 다음 기능을 갖는다.

| 기능 | 의미 |
|---|---|
| 생성 | 스토리지 장치 사용 개시 또는 조건 만족 시 생성 |
| 부여 | 메모리 블록 클로즈 시 메타 데이터에 기록 |
| 갱신 | 특정 블록에 부여된 후 다음 값으로 변경 |
| 비교 | 현재 지시자와 과거 지시자의 간격을 비교 |
| 판단 | 간격이 기준값 이하이면 최근 소거 블록으로 판단 |
| 제어 | 최근 소거 블록이면 라이트 대상에서 제외하거나 재선택 |

이처럼 상태값의 생성-부여-갱신-비교-제어가 시간순으로 이어지기 때문에, 이 문서는 동작-시간형으로 볼 수 있다.

## 6. 도 4~11: 소거 간격 관리 시나리오

### 원문 위치

도 4 내지 도 11 설명 구간.

### 원문 요지

메모리 블록 풀에는 복수의 블록이 있고, 각 블록이 클로즈될 때 소거 지시자 `EI`가 순차적으로 부여된다. 예를 들어 `BLK1`에는 `EI=1`, `BLK2`에는 `EI=2`가 부여되고, 지시자는 계속 갱신된다. 이후 라이트 요청 처리 시 선택된 블록의 과거 지시자와 현재 지시자의 간격을 비교하여 최근 소거 여부를 판단하고, 필요하면 메모리 블록을 재선택한다.

### 분석

이 구간은 흐름도 단계가 아니라 개념 예시다. 그러나 실제로는 본 발명의 핵심 알고리즘을 가장 상세히 설명한다.

전개는 다음과 같다.

| 도면 | 설명 내용 | 전개 기능 |
|---|---|---|
| 도 4 | 초기 메모리 블록 풀과 `EI=1~6` 부여 | 소거 지시자 부여 원리 설명 |
| 도 5 | 사용 중인 `BLK7` 클로즈 시 `EI=7` 부여 | 갱신 흐름 확장 |
| 도 6 | `BLK7` 선택 후 `EI=7`과 현재 `EI=8` 비교 | 최근 소거 여부 판단 |
| 도 7 | `BLK7` 대신 `BLK2` 재선택 | 재선택 제어 설명 |
| 도 8 | `EI=2`와 현재 `EI=8` 간격이 기준값 초과 | 소거 허용 조건 설명 |
| 도 9~10 | 이후 블록 클로즈 및 지시자 갱신 | 라운드-로빈 갱신 지속 |
| 도 11 | 지시자가 없는 `BLK8` 선택 | 예외 상태 처리 |

이 구간은 독자가 추상적인 `소거 체류시간 확보` 개념을 실제 블록 상태 변화로 이해하게 만드는 역할을 한다.

## 7. 도 12: 초기 소거 지시자 생성 흐름

### 원문 위치

도 12 설명 구간, `S101`~`S107`.

### 원문 요지

스토리지 장치의 사용이 개시되면 메모리 컨트롤러가 소거 지시자를 생성한다. 이후 클로즈 대상 메모리 블록이 검출되는지 판단하고, 검출되면 현재 소거 지시자를 메타 데이터로 부여한 뒤 소거 지시자를 갱신한다.

### 단계 구조

| 단계 | 내용 |
|---|---|
| S101 | 소거 지시자 생성 |
| S103 | 클로즈 대상 메모리 블록 검출 여부 판단 |
| S105 | 현재 소거 지시자를 클로즈 대상 블록 메타 데이터로 부여 |
| S107 | 소거 지시자 갱신 생성 |

### 분석

도 12는 도 4~11에서 설명한 개념을 절차화한다. 문장 패턴은 전형적인 흐름도 밀착형이다.

- `메모리 컨트롤러(210)는 ... 할 수 있다(S101)`
- `... 검출되는지 판단할 수 있다(S103)`
- `검출되지 않는 경우(S103:N)`
- `검출되는 경우(S103:Y)`

즉 조건 판단, Y/N 분기, 반복 모니터링을 명시한다.

## 8. 도 13: 조건부 소거간격 관리 모드 흐름

### 원문 위치

도 13 설명 구간, `S121`~`S133`.

### 원문 요지

메모리 컨트롤러는 가용 블록 수를 모니터링하고, 가용 블록 수가 임계값 이하이면 소거간격 관리 모드를 인에이블한다. 이후 소거 지시자를 생성하고, 클로즈 대상 블록이 검출될 때마다 지시자를 부여하고 갱신한다.

### 단계 구조

| 단계 | 내용 |
|---|---|
| S121 | 가용 블록 수 모니터링 |
| S123 | 가용 블록 수가 임계값 이하인지 판단 |
| S125 | 소거간격 관리 모드 인에이블 |
| S127 | 소거 지시자 생성 |
| S129 | 클로즈 대상 메모리 블록 검출 여부 판단 |
| S131 | 소거 지시자 부여 |
| S133 | 소거 지시자 갱신 생성 |

### 분석

도 13은 도 12의 변형 흐름이다. 도 12가 “사용 개시 시점” 중심이라면, 도 13은 “가용 블록 수가 임계값 이하인 조건” 중심이다.

이 구간은 동작-시간형 전개에서 자주 보이는 변형 패턴이다.

> 기본 흐름 제시 → 조건부 변형 흐름 제시 → 동일한 핵심 동작 반복

### 원문 위치

도 14 설명 구간, `S201`~`S215`.

### 원문 요지

라이트 요청이 발생하면 메모리 컨트롤러는 가용 메모리 블록을 선택하고, 해당 블록의 소거 체류시간이 확보되는지 판단한다. 확보되지 않으면 블록을 재선택하고, 확보되면 블록을 소거하여 오픈 상태로 전환한 뒤 라이트 요청을 처리한다. 이후 오픈 블록이 클로즈 대상인지 모니터링하고, 클로즈 대상이면 클로즈 및 지시자 갱신을 수행한다.

### 단계 구조

| 단계 | 내용 |
|---|---|
| S201 | 가용 메모리 블록 선택 |
| S203 | 소거 체류시간 확보 여부 판단 |
| S205 | 가용 메모리 블록 재선택 |
| S207 | 선택된 블록 소거 및 오픈 상태 전환 |
| S209 | 라이트 요청 처리 |
| S211 | 오픈 블록이 클로즈 대상인지 모니터링 |
| S213 | 해당 메모리 블록 클로즈 |
| S215 | 오픈 상태 유지 및 모니터링 반복 |

### 분석

도 14는 이 문서에서 가장 직접적으로 방법 청구항과 대응되는 부분이다. `선택 → 판단 → 재선택/진행 → 처리 → 모니터링 → 클로즈`의 시간 흐름이 명확하다.

특히 `S203`의 판단 결과에 따라 `S205`로 되돌아가거나 `S207`로 진행하는 분기 구조가 있다. 이 때문에 이 문서는 단순한 기능 설명이 아니라 알고리즘형 동작 설명에 가깝다.

## 10. 말미 면책

### 원문 위치

전문 마지막 문단.

### 원문 요지

실시예들은 예시적인 것이며 한정적인 것이 아니고, 본 발명의 범위는 상세한 설명이 아니라 특허청구범위에 의해 정해진다고 설명한다.

### 분석

말미는 표준적인 권리범위 면책 문구다. 별도 변형예를 길게 제시하지 않고, 본문에서 제시한 실시예를 예시로 한정하지 않는다는 방식으로 마무리한다.

## 전개방식 판정

이 문서는 다음 유형으로 정리할 수 있다.

> 블록도 선행 + 상태값 시나리오 + S단계 흐름도형

조금 더 구체적으로는 다음과 같다.

> 컨트롤러 상태관리 로직 전개형

그 이유는 다음과 같다.

| 판단 요소 | 근거 |
|---|---|
| 동작 주체 명확 | 메모리 컨트롤러와 프로세서를 먼저 정의 |
| 시간 흐름 존재 | 소거 지시자의 생성, 부여, 갱신, 비교가 순차적으로 진행 |
| 조건 분기 존재 | `S103:Y/N`, `S123:Y/N`, `S203:Y/N`, `S211:Y/N` 구조 |
| 상태값 중심 | `소거 지시자(EI)`와 메모리 블록 상태가 핵심 |
| 흐름도 밀착 | 도 12~14에서 S단계별 문장으로 방법을 풀어씀 |
| 물리 공정 아님 | 처리 대상은 기판/소재가 아니라 메모리 블록 상태와 데이터 요청 |

## 프롬프트화 가능한 작성 지침

```text
발명을 실시하기 위한 구체적인 내용은 다음 순서로 작성한다.

1. 전체 시스템 구성을 먼저 설명한다.
2. 방법을 수행하는 컨트롤러 또는 장치의 내부 구성을 설명한다.
3. 핵심 동작을 수행하는 기능블록을 기능별로 정의한다.
4. 핵심 상태값 또는 관리값을 정의하고, 그 값이 생성·부여·갱신·비교되는 원리를 설명한다.
5. 복수 도면을 사용해 상태값 변화 시나리오를 예시로 설명한다.
6. 이후 흐름도를 참조하여 S단계 번호별로 방법을 정리한다.
7. 각 단계는 “주체가 무엇을 판단/생성/갱신/제어하는지” 중심으로 쓴다.
8. Y/N 분기가 있으면 조건, 분기 결과, 반복 위치를 명시한다.
9. 마지막에는 실시예 한정 배제 문구를 둔다.
```

## 한줄 요약

이 문서는 메모리 컨트롤러의 구성과 기능블록을 먼저 세운 뒤, `소거 지시자(EI)`라는 상태값의 시간적 변화를 도면 예시와 S단계 흐름도로 반복 설명하는 **동작-시간형 컨트롤러 로직 전개 사례**다.
