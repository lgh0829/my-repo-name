# 명세서 본문 스타일 클러스터링 v2 — JP 마커 출원 제외 기준

> **목적**: JP 번역체 출원(실시'형태' 마커 보유)을 분석 대상에서 제외했을 때의 본문 스타일 클러스터 재정의
> **배경**: v1(`body_style_clustering.md`)은 "JP 여부"가 가장 강한 변별축이라, 그 아래 국내 출원 간 차이가 한 덩어리(C2)로 눌려 있었다. JP를 빼면 숨어 있던 2차 축이 드러난다.
> **데이터**: 전체 3,390건 중 **JP 마커(실시'형태' 사용) 1,002건 제외 → 2,388건** 분석
> **분석일**: 2026-06-05 | 출처: kr_pub_patents + kr_opn_patents (ES)

---

## 0. 핵심 결론

1. **주 변별축이 이동한다: "JP 여부" → "선행기술 인용 + 과제 제기 어법".**
   - JP 제외 후 변별력 순위: **선행기술 인용(41) ≫ 과제 직접서술(9) > 용어정의(8)** > 나머지
   - JP를 빼도 **선행기술 특허문헌을 인용하는 비(非)JP 출원군이 별도로 존재**한다(주로 김앤장 의료·화학·G06F + jang 잔여). 실시'형태'(JP 직역)와 선행기술 인용은 항상 같이 가지 않는다.

2. **v1의 C2(국내 표준)가 분해된다.** JP에 눌려 한 덩어리였던 leeandmok·김앤장이 **과제 어법(직접/간접)으로 양분**된다.

3. **K-means는 이 데이터에서 불안정**(seed 고정에도 군집이 흔들림)하므로, v2는 **재현 가능한 규칙 기반**으로 클러스터를 정의한다. 프롬프트 구현·앱 적용에도 규칙 기반이 적합하다.

## 3. 클러스터별 상세

### A 선행인용형 (474건, 20%)
- **특성**: Background에서 선행기술 특허문헌을 명시 인용(100%). JP 실시'형태'는 없지만 인용 관행은 공유 → **미국 우선권 SW(G06F)·의료(A61)·화학(C) 출원**에서 나타남(CPC A·G·C 집중).
- **사무소**: 김앤장 62% + jang 잔여 19%(실시형태 없이 인용만 하는 jang 출원) + leeandmok 19%
- **주요 대리인(김앤장)**: 김현진(137), 이승현(66), 이재훈(22)
- **대표 출원**: `1020220131642`, `1020230076077`
- **해석**: v1에서 jang의 C1과 국내 일부에 흩어져 있던 "인용하는 출원"이 하나로 결집. **선행기술 인용은 JP 전용 특성이 아니다**는 것을 보여주는 군집.

### B 직접과제형 (587건, 25%)
- **특성**: 과제를 "~하는 문제가 있었다"로 **직접 서술(100%)** + 선행기술 인용 없음 + 배경기술 긺(1,149자)
- **사무소**: 김앤장 52% + leeandmok 47% (거의 균등)
- **주요 대리인(김앤장)**: 김성환(101), 김성남(80), 김기호(43) — 기계·건설(B·H) 비중 높음
- **대표 출원**: `1020240087141`, `1020240194093`
- **해석**: v1의 C4(배경기술 중심·김성환류)가 여기로 흡수·확장. 문제를 명확히 진술하고 배경기술을 충실히 쓰는 국내 전형.

### C 간접·표준형 (1,129건, 47% — 최대)
- **특성**: 과제를 직접 단정하지 않음(간접/중립) + 인용 없음 + **실시예 최장(20,593자)** + 배경기술 짧음(682자)
- **사무소**: 김앤장 48% + leeandmok 48% (균등)
- **주요 대리인(김앤장)**: 김성남(396), 김성환(47), 김지우(18)
- **대표 출원**: `1020220064357`, `1020210178881`
- **하위 구분 가능**: 용어정의 보일러플레이트 보유 17%(197건) → C-a(용어정의 삽입) vs C-b(미삽입)로 세분 가능
- **해석**: v1 C2의 핵심부. 배경기술은 간결히, 실시예는 길고 상세하게 쓰는 국내 표준 다수파.

## 5. 프롬프트 구현 관점 (v2)

JP 출원을 다루지 않을 경우, 본문 생성 프롬프트는 아래 3개 스타일로 분기한다(X 제외).

### 스타일 A — 선행인용형
```
- 배경기술: 선행기술 특허문헌을 명시 인용 ("특허문헌 1에는 …이 개시되어 있다")
- 과제: 직접/간접 혼용 (인용한 선행기술의 한계를 지적)
- 효과: 기재
- CPC 적합: G06F(SW)·A61(의료)·C(화학) — 우선권/인용 관행 있는 분야
```

### 스타일 B — 직접과제형
```
- 배경기술: 충실히 서술(~1,150자), 선행기술 인용 없이 기술 문제만
- 과제: "~하는 문제가 있었다" 직접 단정
- 실시예: 표준 분량(~13,000자)
- CPC 적합: 기계·건설(B·H·E)
```

### 스타일 C — 간접·표준형 (다수파)
```
- 배경기술: 간결(~680자)
- 과제: 단정 회피, 간접/중립 ("~가 요구된다", "~할 필요가 있다")
- 실시예: 장문 상세(~20,000자), 도면 순차 전개
- (옵션) 용어정의 보일러플레이트 삽입 여부로 C-a/C-b 분기
```

## 8. [직교 축] 전개 앵커 — 도면-공간 / 동작-시간 / 실험-수치

> A·B·C 클러스터(선행인용·과제어법 축)와 **독립적인 직교 축**. 실시예 본문이 "무엇을 중심으로 전개되는가"를 가른다. 특히 도면 없는 화학·바이오 출원(A·C)까지 포괄한다.

### 측정 규칙 v2.1 (보정 — 점수 최댓값으로 분류)

> **보정 경위**: v2.0 측정식은 **방법 발명을 도면-공간으로 과대판정**하는 편향이 있었다. 챔버 등 장치 도면(구성도)이 첨부되면 도면수·괄호부호가 많아져, 본질이 공정 단계(동작)·실험인 출원도 도면-공간으로 분류됐다. (PDF 5건 실측에서 기판처리·박막증착 발명이 잘못 분류됨을 확인). 아래는 **S단계·흐름도·방법명칭(→동작), 그래프·SEM·표·비교예(→실험) 신호를 가중**한 보정식이다.

```
도면-공간 = (도면 5장↑?1.0:도면수×0.2) + min(괄호부호/문장,3)×0.6 + min(도N참조/문장,1)×1.0
            (도N 가중 1.5→1.0 하향: 장치도면 과중 억제)
동작-시간 = min(동작·시간어/1000자,10)×0.6 + min(S단계마커/1000자,5)×0.5
            + (도면설명에 흐름도·플로우차트·블록도 ? 0.8 : 0)
            + (발명명칭에 "방법/구동방법" 포함 ? 1.0 : 0)
            (S단계마커 = S10·S101 등 \bS\d{1,3}\b)
실험-수치 = min(실험어/1000자,10)×0.7 + min(수치한정/1000자,10)×0.4 + (도면 1장↓?1.5:0)
            + (도면설명에 그래프·SEM·사진·모니터링·스펙트럼 ? 0.8 : 0)
            + (본문에 "비교예" ? 0.5 : 0) + (표 1/[표 ? 0.5 : 0)
→ 세 점수 중 최댓값을 앵커로. 점수차 작으면 "혼합형"
```
