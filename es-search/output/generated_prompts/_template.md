## [공통 문체 블록] — 5건 전부 적용 (재현 대상)

```
[문체 — 국내 표준형(C 간접·표준형) 고정]
- 실시 용어: '실시예'만 사용 ('실시형태' 금지)
- 실시예 구조: 단일 흐름 (제1/제2 실시예로 쪼개지 말 것)
- **번호 목차·소제목 금지**: "1. 시스템 전체 구성", "2.1. …" 식 보고서형 번호 목차를 쓰지 말 것.
  도면 단위로 문단을 나누되 "도 N을 참조하면 …"으로 자연스럽게 연속 단락 서술한다(한국 명세서 관행).
- **청구항 용어 그대로 유지**: 청구항에 쓰인 용어를 재명명하지 말 것.
  (금지 예: 청구항 "마이크로 프로세서"를 본문에서 "제어부"로 바꿔 부르는 것)
- 용어 처리: 영문 약어는 "한글(영문 풀네임)" 인라인 병기 (예: "HF(high frequency)").
            별도 용어정의·"포함은…"식 해석 보일러플레이트 넣지 말 것
- 도면 참조: 괄호형 부호 `구성요소(110)` 기본 + 문단 도입에 "도 N을 참조하면" 혼용
- 과제 어법: 간접/중립 ("~가 요구된다")
- 효과는 실시예 본문에 녹이지 말 것 ([발명의 효과] 별도 섹션 전제)
- 권리범위 면책: 아래 유형 중 하나를 선택해 1문단 포함 (위치·문구는 {example}의 스타일을 따름)
  [어두 일반] "본 발명의 이점 및 특징, 그리고 그것들을 달성하는 방법은 첨부되는 도면과 함께…청구항의 범주에 의해 정의될 뿐이다."
  [어두 용어] "특별한 정의가 없는 한 본 명세서의 모든 용어는…일반적 의미와 동일하고…어떤 구성요소를 '포함'한다고 할 때…더 포함할 수 있는 것을 의미한다."
  [말미 간결] "이상 본 발명을 바람직한 실시예를 들어 상세하게 설명하였으나, 본 발명은 상기 실시예에 한정되지 않고…변형이 가능하다."
  [말미 청구범위형] "본 발명의 범위는…후술하는 특허청구범위에 의하여 나타내어지며, …등가개념으로부터 도출되는 모든 변경 또는 변형된 형태가 본 발명의 범위에 포함되는 것으로 해석되어야 한다."
  ※ {example}이 없으면 [말미 간결]을 기본으로 사용한다.
```

## [내용 충실도 규칙] — 모든 앵커 공통 (가장 중요)

```
[내용 충실도 — 청구항을 도면·장치 구성으로 완전히 구현할 것]

(F1) 청구항 구성요소 전수 반영:
  · {claim_text}에는 독립항+종속항 전체가 입력된다. 모든 청구항의 구성요소가
    실시예의 장치 구성 및 도면 참조 안에 빠짐없이 포함되어야 한다(뒷받침 요건).
  · 종속항에만 등장하는 구성(예: 유량제어밸브, 버퍼탱크, 스크린필터 등)도 실시예에 반드시 기술한다.
  · 청구항 전체를 기준으로, 도면 대상이 되는 장치를 머릿속에 한 번 구성한 뒤 서술한다.

(F2) "측정·감지 → 그 다음 동작"을 반드시 기술:
  · 청구항에 센서·측정·감지 구성이 있으면, 측정값으로 "무엇을 하는가"(제어·판단·대응)를
    구체적으로 이어서 기술한다. 측정 구성만 나열하고 그 활용(제어 루프·임계 판단·대응 동작)을
    빠뜨리지 말 것.
  · 예) "유량 압력 센서가 압력을 측정한다"에서 끝내지 말고 →
        "측정된 압력이 기준치를 초과하면 노즐 막힘으로 판단하고, 제어부가 해당 노즐에
         세정액을 공급하거나 유량 제어 밸브를 조절한다" 까지 인과·제어 흐름을 서술.
  · 통상의 기술자가 그 발명을 재현할 수 있을 만큼 작동 원리·제어 방식을 구체화한다(실시가능 요건).

(F3) 도면설명에 암시된 구성도 반영:
  · 도면의 간단한 설명에 등장하는 구성(예: "도 N은 ~ 분해 사시도")은 그 내부 구성요소까지
    합리적으로 전개한다. 도면설명에 있는 도면을 본문에서 누락하지 말 것.

(F4) 부호 일괄 부여·일치(앵커 A 규칙과 동일):
  · 구성요소가 처음 등장할 때 부호를 일괄 부여하고, 도면 부호 체계와 일치시킨다.
    (제공된 '구성요소 참조번호 목록'이 있으면 그 번호를 그대로 사용)

⚠️ 단, 청구항·도면·제공 자료에 없는 수치·실험데이터는 임의 생성하지 말 것(환각 금지).
   구조·제어 흐름은 통상 기술 범위에서 합리적으로 보강하되, 정량 수치는 발명자 제공값만 사용.
```

## [앵커 블록 A] — 도면-공간 (구조 발명: 출입문·CMP 등 / H·B·E·F)

```
[전개 — 도면-공간]

(1) 면책 문구 — 위치와 문구는 {example}의 스타일을 따른다 (1문단, 중복 금지):
  · {example}이 어두(도입부)에 면책을 배치하면 → 실시예 첫 문단에 배치
  · {example}이 말미에 면책을 배치하면 → 실시예 마지막 문단에 배치
  · {example}이 없으면 → 말미에 간결형 1문단 사용:
    "이상 본 발명을 바람직한 실시예를 들어 상세하게 설명하였으나, 본 발명은 상기 실시예에
     한정되지 않고, 본 발명의 기술적 사상의 범위 내에서 당 분야에서 통상의 지식을 가진 자에
     의하여 여러 가지 변형이 가능하다."
  · 위치와 무관하게 반드시 1문단만, 동일 내용 중복 금지.
  · 반도체 층/단면 구조일 때만 "도면에서 층 및 영역들의 크기…과장된 것일 수 있다" 추가 가능.

(2) 도면 도입 설명: 각 '도 N'을 처음 언급할 때, 그 도면의 간단한 설명 문단을 1줄 먼저 넣는다.
    예) "도 1은 본 발명의 일 실시예에 따른 [발명명칭]의 개략적인 사시도이다."

(3) 구조도 구성 전개 (도 N이 구조도이고 구성이 처음 등장할 때):
  · [구성 나열 문단] "도 N을 참조하면, [발명품](100)은 A(10), B(20), C(30)를 포함할 수 있다."
  · [구성별 설명 문단] 각 구성마다 한 문단씩 작성. 한 구성을 처음 언급한 뒤 다시 언급할 때는 '상기'를 붙인다.
    예) "상기 A(10)는 상기 B(20)의 하부에 배치될 수 있다. A(10)는 회전축(10a)에 의해 회전될 수 있다.
         예를 들어, 상기 A(10)는 제1 방향(d1)으로 회전될 수 있다."

(4) 연속 참조: 같은 도 N을 다음 문단에서 이어서 참조할 때는 '도 N을 참조하면'을 반복하지 않는다.
    (다른 도면으로 바뀔 때만 "도 M을 참조하면"을 명시)

(5) 복합 도면 처리: 도면이 [대표 구조도 + 사시도/상세도 + α]로 구성된 경우,
    대표 구조도를 위 (3)으로 상술한 뒤, 사시도·상세도 등 나머지 도면은 한 문단으로 묶어 설명한다.

- 구체 수치한정은 최소 (구조 관계 중심)
- 목표 분량: {output_length}자 이상
  ※ {output_length} 결정 규칙:
    · 기준값: **8,000자** (±20% → 6,400자~9,600자 허용 범위)
    · 청구항 수에 관계없이 8,000자를 기본으로 사용한다.
    · 운영 파이프라인: 코드 하드코딩 700 (청구항 1개씩 처리 — 별도 개선 필요)
    · 실행형 프롬프트 파일에는 숫자가 아닌 {output_length} 플레이스홀더를 유지한다.
      (숫자로 굳히지 말 것)
```

## [앵커 블록 B] — 동작-시간 (알고리즘·방법 발명: 데이터처리 등 / G)

```
[전개 — 동작-시간]
- 구성요소 소개 후, 동작을 "먼저 … 이후 … 이어서 …" 시간순으로 전개

- 권한·데이터 흐름을 단계마다 값/주체와 함께 기술
- 목표 분량: 약 17,000자 (장문 — 섹션 분할 생성 권장)
```

## [앵커 블록 C] — 실험-수치 (공정·조성 발명: 기판·박막 등 / C·H 공정)

```
[전개 — 실험-수치]
- 장치·공정 단계(S10~) 기술 후, 공정 조건을 구체 수치범위로 한정
  (예: "HF 파워 650 내지 750W", "압력 2.5 내지 3.5 torr", "타겟 두께의 10 내지 12%")

- 목표 분량: 8,000~13,000자
⚠️ 수치·실험데이터는 발명자 제공값만 사용. LLM 임의 생성 금지(환각 위험).

## [P_REF] 구성요소·부호 매핑 프롬프트 (P9 직후 / P8 직전 실행)

> **목적**: 청구항과 도면의 간단한 설명을 기반으로 각 도면에 등장할 구성요소를 예측하고,
> 일관된 참조번호를 부여한다. P8 Spec Embodiment의 `{drawings}` 입력에 이 결과를 포함시킨다.
>
> **실행 조건**: 구성요소 참조번호 목록(§3)이 없는 경우(신규 발명) 또는 기존 목록과 청구항이 불일치하는 경우.
> §3 목록이 있는 경우에는 해당 목록을 그대로 주입하는 것이 우선.

```
[역할]
너는 특허 명세서 작성 전문가다.
청구항에 기재된 구성요소와 도면의 간단한 설명을 보고,
각 도면에 등장할 구성요소를 예측하고, 전체 발명에 걸쳐 일관된 참조번호를 부여한다.

[입력]
[청구항 전체]: '{claim_text}'
[도면의 간단한 설명]: '{drawings}'

[작업 규칙]
1. 청구항 전체(독립항+종속항)에서 구성요소를 모두 추출한다.
2. 도면의 간단한 설명("도 N은 ~ 사시도/구성도/단면도…")을 읽고,
   각 도면에 어떤 구성요소가 등장할 가능성이 높은지 예측한다.
3. 발명 전체 구성요소에 일관된 참조번호를 부여한다.
   · 주 발명품(최상위 시스템/장치)은 100번대 대표 번호 부여
   · 하위 구성요소는 10·20·30… 또는 110·120·130… 등 계층 반영
   · 번호는 중복 없이, 하나의 구성요소 = 하나의 번호
4. 아래 형식으로 출력한다.

[출력 형식]
<구성요소_부호_목록>
(부호) 구성요소명 — 등장 도면: 도N, 도M…
(부호) 구성요소명 — 등장 도면: …
…
</구성요소_부호_목록>

<도면별_구성요소>

## 검증 체크리스트 (생성 실시예 ↔ 원본 13차원 대조)

**공통 (5건 전부)**
- [ ] '실시예' 사용, '실시형태' 미등장
- [ ] 단일 실시예 흐름 (제N실시예 분할 없음)
- [ ] **번호 목차·소제목 없음** (연속 단락 서술)
- [ ] **청구항 용어 그대로 유지** (재명명 없음, 예 마이크로프로세서→제어부 금지)
- [ ] 영문 약어 인라인 병기, 해석 보일러플레이트 없음
- [ ] 괄호부호 + "도 N 참조" 혼용 / 부호 일괄 부여·도면 일치
- [ ] 효과 별도 섹션 전제, 말미 면책 1문단
- [ ] **<생성_메타> 블록 존재** — 모델명·입력/출력/총 토큰 기록 확인

**내용 충실도 (F1~F4)**
- [ ] F1 전체 청구항(독립항+종속항) 모든 구성요소가 본문·도면에 반영 — 종속항 전용 구성(밸브·탱크·필터 등) 누락 없음

- [ ] F3 도면설명에 등장한 도면·구성 누락 없음
- [ ] F4 부호 일괄 부여·도면 부호 일치 / 임의 수치 환각 없음
