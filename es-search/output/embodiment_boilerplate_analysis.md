# Embodiment 관용문단(면책·방어 단락) 분석 리포트

**분석 일자**: 2026-06-11  
**분석 대상**: `patents_local.db` — `patents_embodiment` 테이블, 3,192건 (Embodiment 500자 이상)  
**목적**: 명세서 Embodiment 섹션에서 반복적으로 등장하는 면책·방어 **관용문단** 수집 및 통계화

> **분석 단위 주석 (현재 한계)**: 현재 DB(`patents_local.db`)의 `Embodiment` 필드는 단락(paragraph) 구분 없이 전문이 하나의 텍스트로 저장되어 있다. 이 때문에 현 버전은 정규표현식으로 관용문단의 **앵커 문구**를 감지하는 방식을 쓴다. `src_db`에 단락 구분 기능이 추가되면 `[NNNN]` 단락 경계를 직접 인식해 관용문단 전체를 정확하게 추출할 수 있다.

## 0. 관용문단이란

관용문단은 Embodiment 섹션 내에서 **하나의 단락(bracket 번호 단위)**으로 등장하는 법적 면책·방어 블록이다. 개별 문장이 아닌 단락 전체가 하나의 법적 목적을 수행한다.

**리앤목 스타일 예시** (1020220060454A, [0026]~[0030], [0097]):

| 단락 번호 | 관용문단 유형 | 법적 목적 |
|---|---|---|
| [0026] | 실시예 면책 + 선행기술 생략 선언 | 균등론 확보, 기재불비 방어 |
| [0027] | 서수 용어(제1·제2) 정의 | 명확성 요건 |
| [0028] | 용어 면책 + 단복수 포함 + 도면 과장 복합 | 명확성·실시 가능 복합 |
| [0029] | 상/하 방향 기준 정의 | 구성요소 위치 기술 명확성 |
| [0030] | 도면 참조 + 동일 도면번호 + 중복 생략 | 실시 가능 요건, 편의 |
| [0097] | 변형·균등 가능 선언 + 청구범위 기준 | 권리범위 해석 분쟁 대비 |

> **단락 내 복수 문장**: 하나의 관용문단이 2~4개 문장으로 구성되는 경우가 일반적 ([0026]: 3문장, [0028]: 3문장, [0097]: 2문장).

## 요약

Embodiment에 등장하는 관용문단은 **위치(서두 / 용어정의블록 / 중간 / 말미)**와 **목적(실시예 한정 부인 / 도면 면책 / 용어 정의 / 추가 가능성 확보 / 변형 가능성 선언)**으로 분류된다.

- 전체 3,192건 중 **55.7%(1,778건)** 에서 1개 이상의 관용문단 패턴이 탐지됨  
- **가장 흔한 단일 패턴**은 중간부의 `"~더 포함할 수 있다"` 문단 (29.8%), 뒤이어 `"이에 한정되지 않는다"` (25.3%)  
- 말미에는 **`"범위는 특허청구범위에 의함"`** 관용문단 (20.5%)이 가장 지배적  
- 서두 관용문단 블록은 `"이하, 첨부된 도면을 참조하여"` 앵커 포함 단락 변형들이 ~35%  
- 관용문단 0개 그룹(44.3%)은 주로 PCT 번역·일본 패밀리 출원이거나 구형 출원 스타일로 추정

## 5. 인사이트

1. **관용문단은 단락(paragraph) 단위의 법적 블록**: 개별 문장이 아닌 단락 전체가 법적 면책 기능을 수행한다. 단락 경계가 확보되면 문장 매칭 오류(탐지 누락·오탐)가 대폭 줄어든다.

2. **복합 관용문단의 존재**: 하나의 단락에 2~3개의 법적 목적이 결합된다 (T28 = T6+T7+T3, [0030] = S1+T1). 이를 개별 관용어 단위로 쪼개면 단락 경계가 사라져 생성 시 복원이 어렵다. 라이브러리는 단락 단위로 저장해야 한다.

3. **"더 포함할 수 있다" 단락이 가장 보편적** (29.8%). 청구항 개방형 기재와 연동되는 본문 미러링으로, 사무소 불문 광범위 사용.

4. **말미 E1/E5 관용문단은 사실상 표준** (20.5%). "특허청구범위에 의하여 나타내어지며" 이하 문장은 거의 동일 텍스트가 반복된다 — 복사·붙여넣기 수준의 공식화.

5. **서두 관용문단 블록은 사무소 식별자** 역할 가능. 리앤목은 5~6개 관용문단을 서두에 연속 배치(클러스터 A). 단독·소규모 사무소는 1~2개 관용문단(클러스터 B). 삼성·반도체 스타일은 3개(클러스터 D).

6. **T10(방향 기준 정의 단락) 신규 확인**: 반도체·전자 분야 명세서에서 상(on)/하(under) 기준을 별도 단락으로 정의하는 패턴. 3D 적층 구조 설명에 필수. 기존 앵커 패턴에 추가 필요.

7. **44.3% 탐지 없음 그룹**: 패턴 부재가 아닌 앵커 매칭 누락(PCT 번역체, 방언 표현) 가능성. 단락 구분 추가 후 정밀 재분류 필요.
