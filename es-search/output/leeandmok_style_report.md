## 3. 프롬프트 수정 제안

### 3-1. id:2 | Avoidance Strategy — 말미 기본값 명시

**수정 전**
```
청구하고자하는 청구항의 <말미>도 작성해줘.
```

**수정 후**
```
청구하고자하는 청구항의 <말미>도 작성해줘.
말미는 '장치'와 '방법'을 기본 쌍으로 설정하고, 발명의 성격상 필요한 경우에만 '시스템', '프로그램', '기록매체'를 추가해줘.
```

**수정 이유**: 말미 총량 장치(985)·방법(999)이 비슷하나, 출원 단위 분석 결과 전략이 다양함. 장치단독(21%)·방법단독(14%) 합산 35%가 단독 카테고리. 발명 유형을 먼저 파악한 뒤 말미를 결정하는 방식이 더 적합.

---

### 3-2. id:3 | Claims Generation — 복수 독립항 전략 및 연결어 지침 추가

**수정 전**
```
<청구항1>
<청구항1의 종속항1>
... (claim_number - 1개)

을 생성해줘. 각 청구항을 작성할 때, 필요한 구성들을 나열해주고, 각 구성을 구체적이고 유기적으로 만들어줘.
각 구성이 어떻게 형성되는지도 포함해서 작성해줘.
종속항은 독립항과 차이를 추가해서 작성해줘.
누가 보아도 똑같이 따라 할 수 있도록 명확하게 한글로 작성해줘.
```

**수정 후**
```
독립항 구성 전략을 먼저 결정한 뒤 청구항을 생성해줘.

[독립항 전략 선택 기준]
- 물리적 장치/기기 발명이고 동작 방법도 보호가치 있음 → 장치+방법 순수형 (② 패턴)
- 소프트웨어·AI·진단 방법이고 프로그램도 유통 가능 → 3-way (장치+방법+기억매체, ③ 패턴)
- 단일 기기 발명이거나 방법이 자명함 → 장치단독 (⑨ 패턴)
- 공정·처리 방법 중심이고 물건 청구 어려움 → 방법단독 (⑩ 패턴)
- 화학 조성물·소재 등 물건 계열 → 물건단독 또는 물건+물건 (⑧/⑪ 패턴)

결정된 전략에 따라 아래 구조로 생성해줘:
<독립항1>
<독립항1의 종속항들>
[추가 독립항이 있는 경우]
<독립항2>
<독립항2의 종속항들>
...

[공통 작성 지침]
- 독립항은 '장치'와 '방법'을 기본 쌍으로 작성한다. 발명의 성격상 필요하면 '시스템'을 추가할 수 있다.
- 구성요소 연결어는 '포함하는', '을 포함하는', '포함하고', '포함하며' 계열을 사용한다. '구비하는'은 사용하지 않는다.
- 구조적 표현(포함하는, 연결된, 형성된, 배치된)을 우선하고, 기능적 표현(수행하는, 생성하는, 판단하는)은 보조적으로만 사용한다.
- 각 구성이 어떻게 형성되는지도 포함해서 작성해줘.
- 종속항은 독립항에 구체적 수치, 추가 기능, 세부 구조를 부가하여 단계적으로 작성해줘.
- 누가 보아도 똑같이 따라 할 수 있도록 명확하게 한글로 작성해줘.
```

**수정 이유**: 출원 단위 분석 결과 전략이 다양(장치단독 21%, 방법단독 14% 등). '장치+방법 쌍 고정'은 과도한 전제. 발명 유형별로 전략을 먼저 결정하는 구조로 개선. '구비하는'은 3%로 비사용 수준.
