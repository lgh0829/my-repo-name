# 분기 계획 / 백로그 우선순위

**트리거**: "다음 분기 계획", "우선순위 정해줘", 새 Epic·Initiative 설계 시.

---

## 절차

1. **회고 인풋 로드** — 직전 분기 `YYYY-QN-retrospective.md`의 이월 후보·드롭 점검·미결 사항을 가져온다.
2. **OKR 기준** — `2026-OKR.md`의 해당 분기 Objective·Initiative를 기준선으로 삼는다 (없으면 작성부터).
3. **분해** — OKR Initiative → Epic → 수직 슬라이스 이슈. 분해 원칙은 `../design/CLAUDE.md`의 수직 슬라이스(Tracer Bullet) 절을 따른다.
   - 얇게 많이 자른다. 각 슬라이스는 완료 시 독립 데모 가능.
   - HITL(리뷰·승인 필요) / AFK(바로 진행) 구분 명시.
   - 이슈 간 의존성(`Blocked by`) 명시.
4. **우선순위** — 아래 순서.

---

## 우선순위 축 (OKR 기여도 우선)

`JIRA.md` 성과평가 가중치와 동일하게 **OKR 기여도**를 1순위로 둔다.

| 축 | 가중 |
|---|---|
| OKR 기여도 (어떤 KR을 직접 움직이는가) | 최상 |
| 측정 가능성 전제 (지표 트래킹 없이는 outcome KR이 채점불가 → 트래킹 선행) | 최상 |
| 비즈니스 영향 (전환·매출·이탈) | 높음 |
| VoC 근거 (`VoC-분석.md`) | 높음 |
| 의존성 (이 작업이 다른 작업의 블로커인가) | 보통 |

> 측정 인프라(이벤트 트래킹)는 다른 기능보다 **먼저** 마감한다. 측정 없이는 다음 분기도 같은 "[측정불가]" 회고를 반복한다.

- 의존성 순서대로 생성 (블로커 먼저, 실제 이슈 번호를 `Blocked by`에 기재).
