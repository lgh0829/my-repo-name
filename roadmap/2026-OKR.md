## 1분기 OKR (2026 Q1)

### Objective: 검색 품질 및 고도화를 통해 사용자가 원하는 특허를 더 정확하게 찾을 수 있게 한다

---

#### KR 1. 검색 성능 업그레이드

| # | Key Result | 평가 방법 |
|---|-----------|---------|
| 1-1 | 검색 결과에 "왜 이 특허가 나왔는지" 이유 표시 기능 제공 | 기능 배포 여부 |
| 1-2 | "원하는 결과가 안 나와요" 버튼 도입 및 재검색/쿼리 수정 이벤트 수집 | 이벤트 수집 정상 동작 여부 |
| 1-3 | 청구항 / 요약 / 문제 / 효과를 필드별 벡터로 분리 임베딩 적용 | 기존 단일 벡터 대비 검색 품질 지표 향상 |
| 1-4 | Top-K 후보를 재평가하는 리랭킹 모델(또는 LLM) 운영 | 리랭킹 적용 전후 nDCG@K 비교 |
| 1-5 | Recall@K / nDCG@K 기반 검색 품질 평가 체계 수립 | 정기 평가 파이프라인 운영 여부 |
| 1-6 | 검색 쿼리 구조화 체계 개선 완료 | 구조화 쿼리 적용 후 검색 품질 지표 개선 |

---

#### KR 2. 검색 방식 고도화 (1차 검색 결과 기반 신규 분야 추천)

| # | Key Result | 평가 방법 |
|---|-----------|---------|
| 2-1 | 클릭 / 저장 / 청구항 확장 / 체류시간 / 추천 클릭 이벤트 수집 구현 | 이벤트 수집 항목 완비 여부 |
| 2-2 | 사용자 피드백(중요 / 무관 / 도움됨) 수집 API 및 저장 체계 구축 | API 정상 동작 및 데이터 적재 여부 |
| 2-3 | 추천 시 "왜 이걸 추천했는지" 근거 텍스트 1~2줄 제공 | 기능 배포 여부 |
| 2-4 | 사용자 중요도 신호를 추천/후속검색 생성에 우선 반영하는 LLM 프롬프팅 구현 | 중요도 반영 추천 정확도 |
| 2-5 | 무관 표시 방향은 추천에서 제외/감점 처리 적용 | 무관 표시 후 동일 방향 추천 억제율 |

## 2분기 OKR (2026 Q2)

### Objective: 명세서 수정 및 도면 생성 기능을 실사용 가능한 수준으로 완성한다

> 전제: 도면 생성 기능 개발 서버 배포 완료 상태에서 시작

---

#### KR 1. 명세서 수정 기능

| # | Key Result | 평가 방법 |
|---|-----------|---------|
| 1-1 | 명세서 일부 수정 시 연관된 다른 항목도 자동 수정 반영 | 연관 항목 자동 수정 정확도 |
| 1-2 | 특허사무소 스타일별(김앤장 / 리앤목 기준) 명세서 포맷 적용 기능 | 스타일 선택 후 출력 포맷 일치율 |
| 1-3 | 기재불비 사항 자동 체크 기능 제공 | 기재불비 검출 항목 커버리지 |

---

#### KR 2. 도면 생성 기능 — 도면 설명 텍스트 생성

| # | Key Result | 평가 방법 |
|---|-----------|---------|
| 2-1 | 명세서 → 도면별 상세 설명 텍스트(구성요소 · 관계 · 배치 포함) 자동 생성 로직 구현 | 생성 텍스트 품질 내부 리뷰 통과 여부 |
| 2-2 | 구성요소, 참조부호, 관계(연결/포함/배치) 정형화 출력 | 정형화 항목 포함율 |
| 2-3 | 자동 생성된 도면 설명을 미리 보고 수정할 수 있는 프론트 화면 제공 | 기능 배포 여부 |

---

#### KR 3. 도면 생성 기능 — 도면 유형 태그 체계

| # | Key Result | 평가 방법 |
|---|-----------|---------|
| 3-1 | 도면 유형 태그 정의 및 기존 특허 도면 분류 완료 (구성 블록도 / 흐름도 / 구조도 / 단면도 / UI 화면도 등) | 분류 완료 및 태그 체계 문서화 |
| 3-2 | 도면 생성 시 도면 유형 태그 + 상세 설명을 함께 활용하는 백엔드 로직 구현 | 기능 동작 여부 |
| 3-3 | 도면 생성 전 도면 유형 선택/확인 UI 프론트 제공 | 기능 배포 여부 |

|------|------|
