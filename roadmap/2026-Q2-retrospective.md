## 4. Keep / Problem / Try

### ✅ Keep (잘된 것)

1. **AI 코어 완주력** — 검색(재검색 100%)·도면(88%)·청구항(87%) 등 난도 높은 AI 백엔드를 분기 내 완성.
2. **건강한 cycle time** — 중앙값 7일. 작은 이슈는 빠르게 흘렀다.
3. **명확한 Epic 구조** — 17개 Epic으로 작업이 응집. 회고 추적이 가능했던 이유.

### 🔴 Problem (문제)

1. **[1순위] OKR이 측정 불가** — Init 5(이벤트 트래킹)가 미흡하여 KR1~4를 단 하나도 채점할 수 없다. OKR이 "작동 불가" 상태. (`2026-OKR.md`도 "모든 Initiative의 전제"라고 명시했으나 실행 우선순위에서 밀림)
2. **소프트런칭 병목** — 가장 outcome에 가까운 Epic(47%)이 분기 말 QA/진행에 정체. 유료 전환 퍼널 미완성.
3. **데이터 위생 부실** — `로드맵`/`목표` 필드 100% 공란, SP 입력률 45%. → OKR 자동 추적 불가, 정량 회고 신뢰도 저하.
4. **드롭 9건의 패턴** — 인프라(OTel 2)·기재요건 보정(2)에 집중. 관측가능성·신규 AI 라인이 "착수했다 접은" 영역. 스코프 설정 또는 우선순위 판단 재점검 필요.
5. **이윤주 완료율 56%** — 개인 이슈가 아닐 수 있음(블로커·의존성). 1:1 확인 필요.

### 🟡 Try (다음 분기 시도)

1. **Init 5를 Q3 최우선·전제 작업으로 격상** — Funnel 이벤트(가입→검색→청구항→명세서→결제) 수집을 다른 기능보다 먼저 완료. 측정 없이는 Q3도 같은 회고를 반복한다.
2. **Jira 필드 강제** — Epic 생성 시 `로드맵`/`목표`(OKR) 필드 필수화, SP 미입력 이슈 스프린트 진입 금지. → 다음 회고는 자동 집계 가능하게.
3. **소프트런칭을 단일 목표 스프린트로** — 유료 전환에 직결되는 잔여 이슈(QA 5·진행 2)를 Q3 초 first-priority로 마감.
4. **드롭 사유 회고** — OTel·기재요건 보정 드롭 6건의 사유를 기록하고, 이월/폐기 결정.
5. **KR2 목표 수치 확정** — "TBD%"로 남은 KR은 측정 자체가 불가. Q3 시작 시 숫자 확정.
