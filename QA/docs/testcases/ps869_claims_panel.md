## 기능 1 — 청구항 목록 조회

---

### TC-01 — 초안 있는 청구항 패널 전체 구조 확인

**설명**: 청구항이 1개 이상 있는 명세서를 `/document` 페이지에서 열었을 때, 청구항 패널이 정해진 구조(개수 표시·카테고리 구분·레이블·내부링크·액션바)대로 렌더링되는지 확인한다. 이 TC는 이후 모든 기능 TC의 전제 조건이므로 가장 먼저 통과해야 한다.

| 항목 | 내용 |
|---|---|
| 시나리오 | `/document` 페이지에서 청구항이 있는 명세서 열기 |
| 진입조건 | 청구항 1개 이상인 명세서 |
| 예상 결과 | ① 총 청구항·독립항·종속항 개수 각각 표시<br>② 발명 카테고리 섹션("장치 발명 N개 청구항")으로 구분<br>③ 독립항/종속항 레이블 배지 표시<br>④ 종속항에 "← 청구항 N" 내부링크 표시<br>⑤ 하단 액션바에 [독립항 추가][종속항 추가][청구항 삭제] 버튼 표시 |
| 결과 | |
| 메모 | 수정완료 표시 → 재테스트 대상 |

---

### TC-01-01 — 초안 없는 상태의 청구항 패널 초기 진입

**설명**: 아직 청구항이 한 건도 없는 신규 명세서에서 청구항 패널을 열었을 때 빈 화면 대신 카테고리 선택 UI와 기본 청구항 자동 생성이 올바르게 동작하는지 확인한다. 초기 사용자가 처음 마주치는 진입 경험이므로 UX 완성도가 중요하다.

| 항목 | 내용 |
|---|---|
| 시나리오 | 메인화면 > AI 명세서 진입 (청구항 0개 상태) |
| 진입조건 | 청구항이 없는 신규 명세서 |
| 예상 결과 | 발명 카테고리 선택 목록 표시. 카테고리 선택 시 독립항 1개 / 종속항 3개 기본 생성 |
| 결과 | |
| 메모 | 1차 PASS 기록됨 — 재확인 권장 |

---

### TC-02 — 종속항 내부링크 클릭 시 포커스 이동

**설명**: 종속항 레이블 옆 "← 청구항 N" 링크는 인용 관계를 시각적으로 탐색하는 핵심 내비게이션 요소다. 클릭했을 때 해당 청구항으로 즉시 스크롤·포커스 이동이 되어야 하며, 이전에 동작하지 않아 FAIL 처리된 후 수정된 항목이다.

| 항목 | 내용 |
|---|---|
| 시나리오 | 종속항 레이블의 "← 청구항 N" 링크 클릭 |
| 진입조건 | 종속항이 1개 이상 있는 청구항 패널 |
| 예상 결과 | 클릭한 인용 청구항 위치로 패널 스크롤·포커스 이동 |
| 결과 | |
| 메모 | 1차 FAIL → 수정완료 표시 → 재테스트 필수 |

## 기능 2 — 독립항 추가

---

### TC-03 — [독립항 추가] 기본 흐름

**설명**: 하단 액션바의 [독립항 추가] 버튼을 눌렀을 때 카테고리 선택 → 빈 청구항 삽입 → 편집 커서 진입까지 전체 흐름이 끊김 없이 동작하는지 확인한다. 1차 테스트에서 `/document` 페이지 버튼 클릭 시 스크롤이 상단으로 강제 이동하는 버그가 있었고, 수정 후 재검증이 필요하다.

| 항목 | 내용 |
|---|---|
| 시나리오 | 패널 하단 [독립항 추가] 클릭 → 카테고리 선택 |
| 진입조건 | 청구항 패널이 열린 `/document` 페이지 |
| 예상 결과 | ① 카테고리 선택 UI 표시<br>② 선택한 카테고리 하단에 빈 청구항 삽입<br>③ 번호 자동 부여<br>④ 편집 커서 자동 진입<br>⑤ 스크롤이 새 청구항으로 이동 (상단 강제 이동 없음) |
| 결과 | |
| 메모 | 1차 W(스크롤 버그) → 수정완료 표시 → 재테스트 필수 |

---

### TC-04 — 독립항 추가 후 내용 입력 및 저장

**설명**: 추가된 빈 독립항에 실제 텍스트를 입력하고 저장했을 때 내용이 정상 반영되는지 확인한다. 청구항을 추가하는 것만큼이나 내용 편집과 저장이 핵심 흐름이다.

| 항목 | 내용 |
|---|---|
| 시나리오 | TC-03 완료 후 → 새 독립항 텍스트 입력 → 저장 |
| 진입조건 | TC-03 통과 상태 |
| 예상 결과 | 입력 내용이 해당 청구항에 저장되고, 패널 목록 텍스트도 업데이트됨 |
| 결과 | |
| 메모 | 신규 TC |

### TC-05 — 카테고리가 2개 이상일 때 독립항 추가

**설명**: 발명이 여러 카테고리(예: 장치 발명 + 방법 발명)로 구성된 명세서에서 독립항 추가 시, 카테고리 선택 UI가 올바르게 표시되고 선택한 카테고리에만 청구항이 삽입되는지 확인한다. 카테고리 구분이 핵심 구조이므로 경계 케이스 검증이 필요하다.

| 항목 | 내용 |
|---|---|
| 시나리오 | 카테고리 2개 이상인 명세서 → [독립항 추가] 클릭 → 카테고리 선택 |
| 진입조건 | 발명 카테고리가 2개 이상인 명세서 |
| 예상 결과 | 카테고리 선택 UI에 2개 이상 옵션 표시. 선택한 카테고리 하단에만 청구항 삽입 |
| 결과 | |
| 메모 | 신규 TC |

### TC-06 — [독립항 추가] 진입 후 취소

**설명**: 카테고리 선택 UI가 열린 상태에서 취소하거나 외부 영역을 클릭했을 때 청구항 목록이 변경되지 않고 원상복구되는지 확인한다. 실수로 버튼을 누른 사용자가 안전하게 이탈할 수 있어야 한다.

| 진입조건 | 카테고리 선택 UI가 표시된 상태 |
| 예상 결과 | 청구항 목록 변경 없음. UI가 닫히고 원래 상태로 복구 |

## 기능 3 — 종속항 추가

---

### TC-07 — 종속항 추가 (단일 인용)

**설명**: 1개의 청구항을 인용하는 종속항을 추가할 때 어두("제N항에 있어서,") 자동 삽입, 번호 부여, 순번 재정렬이 올바르게 동작하는지 확인한다. 1차 테스트에서 순번 재정렬 후 기존 청구항의 인용어구 정합성이 깨지는 문제가 있었으나, 해당 부분은 고도화 논의 중이므로 현재 구현 범위에서 판정한다.

| 항목 | 내용 |
|---|---|
| 시나리오 | [종속항 추가] → 청구항 1개 체크 → [종속항 생성] |
| 진입조건 | 독립항이 1개 이상 있는 청구항 패널 |
| 예상 결과 | ① 어두 '제N항에 있어서,' 자동 삽입<br>② 부모 항 다음 번호 자동 부여<br>③ 중간 삽입 시 이후 청구항 번호 재정렬<br>④ 스크롤이 새 종속항 위치로 이동 |
| 결과 | |
| 메모 | 1차 W(인용어구 정합성) — 고도화 논의 중, 현재 구현 범위 내에서 판정 |

---

### TC-08 — 종속항 추가 (복수 인용)

**설명**: 같은 카테고리의 청구항 2개를 동시에 선택해 다중 인용 종속항을 생성할 때 어두 형식이 올바르게 삽입되는지 확인한다. 1차 테스트 당시 `/document` 액션바 미구현으로 테스트하지 못했으며, 액션바 구현 후 첫 검증이다.

| 항목 | 내용 |
|---|---|
| 시나리오 | [종속항 추가] → 같은 카테고리 청구항 2개 체크 → [종속항 생성] |
| 진입조건 | 같은 카테고리 청구항이 2개 이상인 패널 |
| 예상 결과 | 어두 '청구항 N항에 있어서,' 형식으로 자동 삽입 |
| 결과 | |
| 메모 | 1차 SKIP(액션바 미구현) → 수정완료 표시 → 재테스트 필수 |

---

### TC-09 — 다른 카테고리 청구항 복수 선택 시도 (제한 검증)

**설명**: 서로 다른 발명 카테고리에 속한 청구항을 동시에 인용하는 종속항 생성은 특허법상 허용되지 않는다. 이를 UI 수준에서 차단하는지 확인한다. 1차 테스트 당시 미구현으로 테스트하지 못했으며, 수정 완료 후 첫 검증이다.

| 항목 | 내용 |
|---|---|
| 시나리오 | [종속항 추가] → 카테고리가 다른 청구항 2개 선택 시도 |
| 진입조건 | 발명 카테고리가 2개 이상인 명세서 |
| 예상 결과 | 다른 카테고리의 청구항은 선택 불가(비활성화) 또는 선택 시 경고 메시지 표시 |
| 결과 | |
| 메모 | 1차 SKIP(액션바 미구현) → 수정완료 표시 → 재테스트 필수 |

| 예상 결과 | 자동 삽입된 어두("제N항에 있어서,") 유지. 커서 위치 어두 뒤. 이후 텍스트 자유 입력 가능 |
| 결과 | |
| 메모 | 신규 TC |

## 기능 4 — 청구항 삭제

---

### TC-12 — 종속항 없는 독립항 단건 삭제

**설명**: 하위 종속항이 없는 독립항을 삭제할 때 경고 모달 → 확인 → 삭제 → 번호 재정렬 흐름이 올바르게 동작하는지 확인한다. 1차 테스트에서 `/document` 페이지 삭제 버튼 클릭 시 스크롤이 상단으로 강제 이동하는 버그가 있었으며, 수정 후 재검증이 필요하다.

| 항목 | 내용 |
|---|---|
| 시나리오 | 종속항 없는 독립항의 [삭제] 버튼 클릭 → 경고 모달 확인 |
| 진입조건 | 하위 종속항이 없는 독립항이 존재하는 패널 |
| 예상 결과 | ① 경고 모달 표시<br>② 확인 클릭 시 해당 청구항만 삭제<br>③ 이후 번호 재정렬<br>④ 스크롤 위치 유지(상단 강제 이동 없음) |
| 결과 | |
| 메모 | 1차 W(스크롤 버그) → 수정완료 표시 → 재테스트 필수 |

---

### TC-13 — 종속항을 가진 독립항 삭제

**설명**: 하위 종속항이 딸린 독립항을 삭제하면 종속항도 함께 삭제된다는 사실을 사용자에게 모달로 고지한 뒤 일괄 삭제가 이루어지는지 확인한다. 사용자가 의도하지 않게 종속항을 날리는 사고를 방지하는 UI 보호 장치 검증이다.

| 항목 | 내용 |
|---|---|
| 시나리오 | 종속항이 있는 독립항의 [삭제] 클릭 → 경고 모달 확인 |
| 진입조건 | 종속항 1개 이상인 독립항이 존재하는 패널 |
| 예상 결과 | ① 경고 모달에 "종속항도 함께 삭제됨" 안내 문구 포함<br>② 확인 시 독립항+종속항 모두 삭제<br>③ 번호 재정렬 |
| 결과 | |
| 메모 | 1차 W(인용어구 정합성) → 수정완료 표시 → 재테스트 필수 |

### TC-14 — 자식 종속항이 있는 종속항 삭제

**설명**: 종속항의 하위에 또 다른 종속항이 있는 중첩 구조에서 상위 종속항을 삭제하면 하위 종속항까지 연쇄 삭제되는지 확인한다. 단순 독립항 삭제(TC-13)보다 복잡한 트리 구조 삭제 케이스다.

| 항목 | 내용 |
|---|---|
| 시나리오 | 하위 종속항이 있는 종속항의 [삭제] 클릭 → 경고 모달 확인 |
| 진입조건 | 종속항의 하위에 또 다른 종속항이 있는 구조 |
| 예상 결과 | ① 경고 모달 표시<br>② 대상 종속항 + 하위 종속항 모두 삭제<br>③ 번호 재정렬 |
| 결과 | |
| 메모 | 1차 W(인용어구 정합성) → 수정완료 표시 → 재테스트 필수 |

### TC-15 — [청구항 삭제] 액션바 — 독립항 선택 시 종속항 자동 선택

**설명**: 패널 하단 [청구항 삭제] 버튼을 눌러 다중 선택 모드로 진입했을 때, 독립항을 체크하면 해당 독립항을 인용하는 종속항이 자동으로 함께 선택되는지 확인한다. 이를 통해 사용자가 의도치 않게 독립항만 삭제하고 고아 종속항을 남기는 상황을 방지한다.

| 항목 | 내용 |
|---|---|
| 시나리오 | [청구항 삭제] 버튼 클릭 → 다중 선택 모드 진입 → 독립항 체크박스 선택 |
| 진입조건 | 종속항이 있는 독립항이 존재하는 패널 |
| 예상 결과 | 선택한 독립항을 인용하는 종속항이 자동으로 함께 선택됨 |
| 결과 | |
| 메모 | 1차 SKIP(액션바 미구현) → 수정완료 표시 → 재테스트 필수 |

### TC-15-01 — [청구항 삭제] 다중 선택 후 취소

**설명**: 다중 선택 모드에서 삭제 대상 청구항을 선택했다가 취소했을 때, 청구항 목록이 변경되지 않고 다중 선택 모드가 완전히 해제되는지 확인한다. TC-15의 안전 이탈 케이스다.

| 항목 | 내용 |

| 진입조건 | 다중 선택 모드 진입 상태 |
| 예상 결과 | 청구항 목록 변경 없음. 다중 선택 모드 해제. 일반 뷰 복구 |

### TC-19 — 기재불비 검증 실행 및 결과 표시

**설명**: AI 코파일럿의 [청구항 기재불비 검토하기]를 실행했을 때 기재불비 항목 목록이 표시되고 해당 청구항 위치로 이동할 수 있는 링크나 하이라이트가 제공되는지 확인한다. 특허 심사 관점에서 문제가 있는 청구항을 빠르게 파악하는 핵심 기능이다.

| 결과 | |
| 메모 | 신규 TC |

### TC-20 — 기재불비 없는 청구항으로 검증 실행
