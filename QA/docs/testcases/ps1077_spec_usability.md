### TC-09 — 더블클릭으로 편집 모드 전환

| 항목 | 내용 |

| 예상 결과 | ① 해당 항목이 편집 모드로 전환됨<br>② 텍스트 편집 커서 활성화<br>③ 완료/취소 버튼(또는 동등한 UI) 표시<br>④ 싱글클릭으로는 편집 모드 진입 안 됨 |
| 결과 | |
| 메모 | 더블클릭 인식 범위(텍스트 영역 전체 vs 일부) 현장 확인 |

### TC-09-01 — 빈 항목 진입 시 식별항목별 플레이스홀더 표시

**설명**: 내용이 없는 항목을 편집 모드로 진입했을 때 각 식별항목에 맞는 안내 플레이스홀더가 표시되는지 확인한다. 모든 항목에 동일한 텍스트가 아닌 항목별로 다른 플레이스홀더를 제공해야 사용자가 입력 맥락을 즉시 파악할 수 있다.

| 항목 | 내용 |
|---|---|
| 시나리오 | 내용이 없는 각 식별항목을 더블클릭해 편집 모드 진입 |
| 진입조건 | 내용이 비어 있는 명세 항목 |
| 예상 결과 | 항목별로 아래 플레이스홀더 텍스트가 표시됨:<br>• 기술분야: "발명의 기술분야를 명확하고 간결하게 입력하세요."<br>• 배경기술: "발명의 이해에 유용한 종래의 기술을 입력하세요."<br>• 해결하려는 과제: "발명이 해결하고자 하는 과제를 입력하세요."<br>• 과제의 해결 수단: "과제를 해결하기 위한 기술적 수단을 입력하세요."<br>• 발명의 효과: "종래 기술 대비 발명의 효과를 입력하세요."<br>• 도면의 간단한 설명: "각 도면 번호와 내용을 입력하세요."<br>• 발명을 실시하기 위한 구체적인 내용: "통상의 기술자가 발명을 재현할 수 있도록 구성·동작·재료 등 실시에 필요한 내용을 입력하세요."<br>• 실시예: "발명의 구체적인 실시 형태를 입력하세요."<br>• 요약: "발명의 기술적 특징과 핵심 내용을 간결하게 입력하세요. (400자 이내)" |
| 결과 | |
| 메모 | 9개 식별항목 각각 확인. 내용 입력 시 플레이스홀더가 사라지는지도 확인 |

### TC-09-02 — 편집 모드 스크롤 시 플로팅 타이틀 표시

**설명**: 내용이 채워진(filled) 항목을 편집 모드로 진입한 상태에서 본문을 세로 스크롤할 때, 현재 편집 중인 식별항목 타이틀과 완료/취소 버튼이 플로팅 형태로 고정 표시되는지 확인한다. 긴 항목을 편집할 때 현재 편집 위치와 저장 버튼을 놓치지 않도록 하는 UX 요소다.

| 항목 | 내용 |
|---|---|
| 시나리오 | 내용이 있는 항목 더블클릭(편집 모드 진입) → 본문 영역 세로 스크롤 |

| 결과 | |
| 메모 | content: empty 상태에서는 플로팅 표시 여부 별도 확인 필요 |
