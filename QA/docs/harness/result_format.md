# 테스트 결과 문서 작성 형식

## 파일 위치 및 이름

`output/<이슈키>_results.md`

예: `output/ps869_results.md`

## 문서 구조

```markdown
# <이슈키> 테스트 결과

**이슈**: [이슈키](Jira URL)
**테스트 일시**: YYYY-MM-DD
**테스터**: 이름
**테스트 환경**: 상용서버 / Chrome

---

## 요약

| 항목 | 수치 |
|---|---|
| 전체 TC | N |
| PASS | N |
| FAIL | N |
| 부분(W) | N |
| SKIP | N |

---

## 기능별 결과

### 기능 N — 기능명

| TC | 결과 | 메모 |
|---|---|---|
| TC-XX | ✅ / ❌ / ⚠️ / ⏭️ | 실제 관찰 내용 |

**스크린샷**: `output/screenshots/<이슈키>_TC-XX_<PASS|FAIL|WARN>.png`

---

## FAIL/W 상세

### TC-XX — 시나리오 제목

- **결과**: ❌ FAIL
- **재현 경로**: 메인화면 > ... > 버튼 클릭
- **예상**: ...
- **실제**: ...
- **스크린샷**: 첨부
- **등록된 버그 이슈**: [PS-XXX](링크)

---

## 최종 의견

테스트 완료 후 관찰된 전반적인 품질, 주의 사항, 후속 조치 제안을 자유 기술.
```

## 스크린샷 명명 규칙

`<이슈키>_<TC번호>_<상태>.png`

예:
- `ps869_TC-01_PASS.png`
- `ps869_TC-02_FAIL.png`
- `ps869_TC-07_WARN.png`
