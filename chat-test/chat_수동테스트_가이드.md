# 채팅 에이전트 수동 테스트 가이드 (팀원 공유용)

> 변리사 보조 채팅(에픽 PS-730)의 dev 환경 테스트 안내입니다.
> 터미널에서 5분이면 시작할 수 있습니다. 문의: 배성현

## 1. 준비물

| 항목 | 내용 |
|---|---|
| 파일 | 전달받은 `chat_repl.py` 1개 (dev 서버 주소·테스트 계정 토큰 내장) |
| Python | 3.10+ 와 `pip install httpx` |

## 2. 실행

```bash
pip install httpx
python chat_repl.py
```

그게 전부입니다 — dev 서버(`data-dev.patsol.kr`)와 공용 테스트 계정으로 바로 붙습니다.

> 모두 **같은 테스트 계정**으로 접속합니다. 대화는 각자 새로 생성되니 섞이지 않지만,
> 적재 가능한 청구항/명세서도 테스트 계정 소유 데이터입니다(문서 ID는 공지된 것 사용).
> 본인 계정으로 테스트하려면: PatSol dev 로그인 → F12 → Network → Authorization 헤더의
> Bearer 토큰 복사 → `python chat_repl.py --token <JWT>`

### REPL 명령

| 명령 | 동작 |
|---|---|
| (그냥 입력) | 메시지 전송, 응답 스트리밍 |
| `/claim 123` | 이후 메시지에 청구항 ID 컨텍스트 포함 (`/claim` 으로 해제) |
| `/doc 45` | 이후 메시지에 명세서(문서) ID 포함 |
| `/new` | 새 대화 시작 |
| `/quit` | 종료 |
| 응답 중 `Ctrl+C` | 그 응답만 중단 (중단 동작 테스트용 — REPL은 유지) |

claim/document ID는 슬랙에 공지된 테스트 데이터 ID를 쓰세요.

## 3. 테스트 시나리오

위에서부터 순서대로 해보세요. 문서/청구항 ID는 공지된 테스트 데이터 ID를 쓰세요. "기대 동작"과 다르면 §4 양식으로 알려주세요.

### A. 기본 대화
| 입력 | 기대 동작 |
|---|---|
| `안녕하세요` | 변리사 보조 에이전트 소개 |
| `오늘 저녁 메뉴 추천해줘` | 특허 업무 외 질문 정중히 거절 |

### B. 데이터 적재 (테스트 데이터 ID로)
| 입력 | 기대 동작 |
|---|---|
| `/doc N` 후 `이 명세서 요약해줘` | `[tool:load_spec 실행 중...]` 표시 후 요약 |
| `이 문서 도면 설명해줘` | `tool:load_drawings` 호출 |
| `/claim N` 후 `청구항 구성 보여줘` | `tool:load_claims` 호출 |
| `저장된 기재불비 있어?` | `tool:load_requirements` 호출 |

### C. 작업 도구
| 입력 | 기대 동작 |
|---|---|
| `청구항 기재불비 정밀 검사해줘` | `tool:review_claim` — **30~50초 걸림** (실행 중 표시 유지), 기계적/의미적 결함 + 심각도 보고 |
| `센서 각도 조절 종속항 하나 써줘` | `tool:write_claim` — 기존 청구항 용어를 따르는 초안 |
| `배경기술 문단 더 간결하게 다듬어줘` | `tool:edit_spec` — 수정안 제시, "원본은 변경 안 됨" 안내 |
| `"상기 장치" 표현의 대안 있을까?` | `tool:suggest_expr` — 대안 3개 + 근거 |

### D. 멀티턴 (대화 기억)
같은 대화에서 연달아:
1. `명세서 N번 기재불비 검토해줘`
2. `두 번째 문제만 자세히 설명해줘` ← 이전 결과를 기억하는지
3. `그걸 고치려면 어떻게 수정해야 해?` ← "그걸"이 통하는지
4. `/new` 후 `아까 뭐 검토했었지?` ← 새 대화에선 모르는 게 **정상**

### E. 중단
- 긴 응답 중 `Ctrl+C` → 중단 후 같은 대화에서 `방금 하던 얘기 이어서 해줘` — 중단 전 내용까지 기억하면 정상

### F. 권한 (이상 동작이 정상인 케이스)
- 남의 문서 ID로 `/doc` 지정 후 적재 요청 → **접근 거부**가 정상 (보안 동작)
- 존재하지 않는 ID → 동일하게 거부

## 4. 문제 보고 양식

이상 동작 발견 시 아래를 함께 공유해 주세요 (슬랙):

```
- 보낸 메시지:
- 기대한 동작:
- 실제 동작: (REPL 출력 복사)
- 대화 ID: (REPL 시작 시 표시되는 conversation_id)
- 시각:
```

> 응답 품질 피드백(말투, 검토 결과의 유용성, 초안 수준)도 같은 양식으로 환영합니다 —
> 프롬프트 개선에 바로 반영됩니다.
